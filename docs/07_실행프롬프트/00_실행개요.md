## 0. 이게 뭔가

docs-em(100% 로컬 한국어 사내문서 RAG)을 **페이즈 단위로 끊어** 개발하기 위한 한방 프롬프트 묶음이다. 각 `NN_페이즈*.md`는 그 자체로 완결된 작업 지시서이며, 한 페이즈를 마칠 때마다 **컨텍스트를 초기화**하고 다음 페이즈를 새로 시작한다.

- **왜 끊는가**: 컨텍스트 누적으로 인한 지시 희석·불변식 망각·환각을 차단한다. 페이즈마다 깨끗한 컨텍스트에 해당 프롬프트 전체를 다시 주입한다.
- **공통 전제**: 모든 페이즈 프롬프트 상단에는 확정 사실(정체·불변식 INV-1~8·확정 스택·함수 계약)이 공통으로 박혀 있다. 이 범위 밖 수치/사실 발명은 금지다.
- **한방성**: 한 페이즈 프롬프트를 붙여넣으면 그 페이즈의 코드·테스트·완료기준 확인까지 한 사이클에 끝난다.

## 1. 사용 절차

각 `NN_페이즈*.md`를 **번호 순서대로** 다음 사이클로 처리한다.

1. **새 컨텍스트 열기** — 직전 페이즈 대화를 닫고 빈 컨텍스트를 연다.
2. **파일 전체 붙여넣기** — 해당 페이즈 `.md`를 통째로 붙여넣는다(상단 공통 사실 포함).
3. **완료기준 통과 확인** — 페이즈 하단의 완료기준(테스트·게이트 수치)을 실제로 실행해 통과를 확인한다. 미통과 시 같은 컨텍스트에서 수정.
4. **커밋** — 통과한 산출물을 커밋한다(페이즈 단위 커밋).
5. **컨텍스트 초기화** — 대화를 닫는다.
6. **다음 페이즈** — 1번으로 돌아가 다음 번호를 연다.

> 게이트 페이즈(4·6·8)는 3번에서 정량 수치를 만족하지 못하면 4번(커밋)·6번(다음 페이즈)으로 **진행 금지**.

## 3. 페이즈 간 의존성

```
0 → 1 → 2 → 3 → 4 ──(L1 게이트)──→ 5 → 6 ──(측정 게이트)──→ 7 → 8 (L3 게이트)
```

- **0**: 모든 페이즈의 선행. Phase 0 실측 수치(norm·차원·reasoning 폴백)가 1·3·6의 가드/계약 근거.
- **1**: 2·3·4·6·7·8 모두 `chat()`/`embed()` 래퍼에 의존. 불변식 경계가 여기서 확정되므로 우회 금지.
- **2 → 3**: 청킹 산출(`Chunk`)이 인덱싱 입력.
- **3 → 4**: 인덱스·헤더가 `search()`/`evaluate()`의 대상.
- **4 → 5**: dense baseline Recall@k가 하이브리드 기여도 비교의 기준선. 골든셋·평가 하네스가 5·6·8 게이트의 측정 도구.
- **5 → 6**: 하이브리드 파이프라인 위에서 임베딩만 단독 교체해 측정.
- **6 → 7**: 채택된 임베딩(nomic 또는 BGE-M3) 위에서 답변 생성.
- **7 → 8**: 검색 결과에 리랭커를 덧대 재정렬. 8은 진입 전 서빙 방식 실측 선행 필수.

## 4. 게이트

게이트는 정량 수치로만 통과 판정한다. 미통과 시 해당 페이즈에서 **진행 금지**(커밋·다음 페이즈 불가).

- **L1 게이트 (페이즈 4) — 회귀 재현**: 회귀 케이스("연차 휴가 신청 절차" → 휴가규정)가 골든셋에 포함되고, 평가 하네스가 현재 dense-only에서 이 케이스 **Recall@1=0**을 재현해야 한다. 0이 정상(아직 못 맞히는 상태를 측정으로 고정). 합격선 = 회귀 케이스가 측정에 잡힘.
- **측정 게이트 (페이즈 6) — BGE-M3 채택**: BGE-M3를 **단독 변수**로 교체(다른 변수 고정)해 골든셋 Recall@1이 nomic 대비 **개선**되어야 채택. 미개선 시 nomic 롤백(768차원 전체 재인덱싱 원복 + norm 재측정). 회귀 케이스 0→1 전환이 1차 성패 척도.
- **L3 게이트 (페이즈 8) — 리랭커**: BGE-reranker-v2-m3 **로컬 서빙 방식 실측**을 진입 전 선행(LMStudio 호환 불확실 → sentence-transformers/FlagEmbedding 별도 프로세스 가능). 리랭킹이 top_n 품질에 기여(Recall/MRR 비퇴행)함을 측정으로 증명해야 통과.

> 공통: INV-7. 정량 증명은 Recall@k(1/3/5/10)·MRR·nDCG@10으로 한다. 변수는 한 번에 하나만 바꿔 측정한다.

## 5. 절대 규칙 요약

- **외부 API 금지**: 모든 추론/임베딩/생성은 로컬 LMStudio(`http://localhost:1234/v1`)만. OpenAI/Anthropic/Cohere 등 외부 클라우드 호출 0건(INV-3).
- **INV-1**: 임베딩 재정규화 금지. nomic 출력은 이미 L2 정규화(norm=1.0). 가드 `abs(norm-1.0)<1e-3`. 모델 교체 시 norm 재측정.
- **INV-2**: `content` 빈값 → `reasoning_content` 폴백. 생성모델 3종 모두 답이 reasoning_content에 옴. `/no_think`·`enable_thinking=false`·`reasoning_effort=low`·`max_tokens=4000` 우회책 전부 무효(코드에 넣지 말 것).
- **INV-3**: 외부 API 금지. base_url localhost:1234 외 호출 0건.
- **INV-4**: `prompt_tokens=0` 불신. usage 오보고 → 길이 판단은 문자 수/로컬 토크나이저 가드로(fail-closed, 무음 통과 금지).
- **INV-5**: `<think>…</think>` 정제·최종답 추출. 구분자 불일치 시 원문+[경고] 반환(무음 빈답 금지).
- **INV-6**: `finish_reason=length`면 max_tokens×3로 딱 1회 재시도, 2회째도 length면 경고 부착 반환(무한루프 금지).
- **INV-7**: 정량 증명. 골든셋 Recall@k·MRR·nDCG로 증명. 회귀 케이스 합격선 Recall@1=1. 변수 1개씩만 변경 측정.
- **INV-8**: prefix 효과 미미(nomic 0.687→0.695). 단 e5 계열 교체 시 `query:`/`passage:` prefix 필수.

### 이 PC 기준 보정 사항 (이미 프롬프트에 반영됨)

- **모델 id prefix**: `qwen/qwen3.5-9b` 등 — LMStudio 실측 id 그대로(prefix 없으면 `chat()` 404).
- **kss 버전 상한**: `kss>=4.5,<5` — 최신 6.x는 mecab/pecab 백엔드가 4.x와 비호환이라 macOS arm64에서 깨짐. 설치 실패 시 정규식 종결어미 규칙으로 폴백.
- **Python 3.11 고정**: 모든 `uv venv`는 `--python 3.11`. 기본 3.14로 만들면 lancedb/kiwipiepy 휠 미존재.
- **lancedb 핀**: `>=0.33,<0.34`(실측 0.33.0). `_distance` 반환범위는 설치 후 버전 확인하고 주석.
- **kiwi 버전 조회**: `importlib.metadata.version("kiwipiepy")`(메타데이터 기반, 가장 robust).

> 참고: macOS의 `grep`은 BSD 계열이지만 `-rnE`·`-rniE`·`\b`를 모두 지원하므로 검증 명령은 그대로 동작합니다(실측 확인됨).
