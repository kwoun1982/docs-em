# 08단계: BGE-M3 임베딩 교체 실험 (측정 게이트)

### 이 페이즈의 목표
현행 nomic 임베딩(768차원)을 BGE-M3(1024차원)로 교체하는 단독변수 실험을 수행하고, 동일 골든셋에서 Recall@1 개선이 숫자로 확인될 때만 채택한다(미개선 시 nomic 인덱스로 롤백). 차원 변경에 따른 전체 재인덱싱과 norm 재측정·metric 재확인을 동반한다.

### 절대 규칙 (이 페이즈에서 위반 금지)
- **INV-3 외부 API 금지**: 모든 임베딩 호출은 `base_url=http://localhost:1234/v1`(더미 `api_key="lm-studio"`)로만. OpenAI/Cohere/HuggingFace Inference 등 외부 호출 0건. BGE-M3도 반드시 로컬 LMStudio GGUF로 서빙.
- **INV-1 재정규화 규칙 변경 주의**: nomic은 이미 norm=1.0이라 재정규화 금지였다. **BGE-M3는 norm=1.0이 보장되지 않으므로** 반드시 **재측정**한다. 가드 `abs(norm-1.0)<1e-3`가 트립하면, nomic 경로는 그대로 두고 **BGE-M3에 한해서만** 정규화 분기를 추가한다. 단 정규화 연산 자체에서 **`norm>0`인 벡터만 `v/norm`을 적용하고, `norm==0`(빈/제로 벡터)이면 정규화를 건너뛰어(0벡터 유지) 0으로 나누기를 방지**한다(빈입력 스킵 + 배치 전체 assert 원칙 유지). 측정 없이 정규화 분기를 넣거나 빼지 말 것.
- **INV-7 단독변수 + 정량 증명**: 이번에 바꾸는 변수는 **임베딩 모델 하나뿐**이다. 청크 크기·top_k·BM25·리랭커·생성모델 등 다른 변수 동시 변경 금지. before(nomic)/after(BGE-M3)는 동일 골든셋·동일 `k_list`로 측정하고, 채택 판정은 숫자로만 한다. 합격선은 회귀 케이스 Recall@1=1 포함.
- **회귀 케이스 1급 시민**: 질의 "연차 휴가 신청 절차" → 정답 휴가규정. nomic dense-only에서 현재 3위(Recall@1=0). 이 **0→1 전환**이 채택의 핵심 척도다. 골든셋에서 빼지 말 것.
- **INV-4 prompt_tokens 불신**: 길이 판단은 문자 수/로컬 토크나이저 가드로(BGE-M3 ctx 8192 — nomic보다 길지만 여전히 `assert_within_embed_limit` 문자 가드 유지, fail-closed).
- 임베딩 교체와 무관한 리팩토링·기능 추가 금지.

### 작업 지시

**0) 모델 카드 확인 (코드 전에)**
BGE-M3 prefix 규칙·정규화 여부·권장 metric은 **[확인필요]** 상태다. LMStudio 모델 카드/공식 문서로 다음을 확정하고 결과를 주석/리포트에 남긴다.
- prefix: **BGE-M3의 prefix 규칙을 모델 카드로 확인한다(불필요할 수 있으나 단정 금지).** e5 계열은 `query:`/`passage:` prefix가 필수(INV-8)이므로 BGE-M3와 혼동 금지 — 단, BGE-M3 자체의 prefix 필요 여부는 모델 카드 실측으로만 결정하고, 결과를 `config/settings.yaml`의 prefix 필드에 명시한다(실측이 "필요"로 나오면 그대로 반영).
- 정규화: 출력 벡터가 L2 정규화되어 나오는지(아래 1에서 실측으로 최종 판정).
- metric: cosine 권장 여부 확인(LanceDB `create_index`에서 명시, 기본 L2 유클리드 금지).

**1) norm 재측정 스크립트** — `scripts/measure_bge_norm.py` (신규)
BGE-M3로 대표 한국어 문장 N개(회귀 케이스 질의·정답 문장 포함)를 임베딩해 차원과 norm 분포를 실측한다. 모델 id는 사전조건 7에서 기록한 실제 `<bge_id>`를 인자/설정으로 주입한다(하드코딩 금지).
```python
# 핵심: localhost:1234만 사용, 모델 id는 <bge_id>(사전조건 7의 실측 id), 결과를 stdout + JSON으로
# - dim 확인 (1024 기대)
# - 각 벡터 L2 norm 계산 → min/max/mean, abs(norm-1.0)<1e-3 통과 비율
#   (norm==0 벡터는 정규화 대상에서 제외 — 0으로 나누기 방지, 별도 카운트)
# - 판정: 통과율 100% → 정규화 분기 불필요 / 아니면 BGE-M3 정규화 분기 필요
```
출력: `dim=...`, `norm min/max/mean=...`, `needs_normalize=True|False`.

**2) `embedder.py` 교체 경로** — `src/embed/embedder.py` (수정)
시그니처 고정, **계약 변경 금지**:
- `embed(texts: list[str], model="text-embedding-nomic-embed-text-v1.5") -> list[Vector]` — 기본값은 그대로 두고, BGE-M3는 호출 시 `model="<bge_id>"`로 전달하거나 `config/settings.yaml`에서 모델명을 읽어 주입.
- 빈입력 → `[]` 즉시 반환(기존 유지).
- norm 가드 유지하되, **모델별 정규화 정책 분기**를 추가: BGE-M3에서 1의 `needs_normalize=True`로 판정된 경우에만 임베딩 후 L2 정규화한 뒤 가드 통과시킨다. 정규화 시 **`norm>0`인 벡터만 `v/norm`을 적용하고 `norm==0`이면 스킵(0벡터 유지)** 하여 0으로 나누기를 방지한다. nomic 경로는 손대지 않는다(여전히 재정규화 금지).
- `assert_within_embed_limit(text, max_chars=DEFAULT_MAX_CHARS_FALLBACK=1000)` 문자 가드 유지(fail-closed).

**3) 설정 스위칭** — `config/settings.yaml` (수정)
임베딩 모델명·`dim`·`norm`·`metric`·`prefix`·활성 `index_dir` 플래그를 BGE-M3 값으로 바꾼 프로파일을 추가한다(nomic 프로파일은 보존해 롤백 가능하게). search/evaluate가 이 활성 설정을 읽어 질의 임베딩 모델·prefix·인덱스를 일관 전환하도록 한다. 예: 활성 임베딩을 `<bge_id>`(사전조건 7에서 기록한 LMStudio 실제 id — 하드코딩 금지), `dim: 1024`, `metric: cosine`, `norm: <1에서 실측>`, `prefix: <0에서 확정>`, `index_dir: data/index_bge`.

**4) 전체 재인덱싱 (768→1024)** — 기존 인덱싱 진입점 사용(없으면 폴백)
차원이 바뀌므로 **인덱스를 새 디렉토리에 새로 빌드**한다(기존 nomic 인덱스 디렉토리는 보존해 롤백 경로 유지).
- `data/index_nomic/`(기존, 보존) ↔ `data/index_bge/`(신규) 분리.
- 진입점: 사전조건 6에서 `src/app/cli.py`에 `reindex`(+`--embed-model`/`--index-dir`)가 **있으면 그것을 사용**한다. **없으면** 인덱싱 모듈(`src/ingest/{loaders,chunker}` + `src/store/vector_store.py`)을 직접 호출하는 폴백 빌드 절차를 작성한다(임베딩 모델 = `<bge_id>`, 출력 디렉토리 = `data/index_bge`).
- LanceDB `create_index`에서 `metric`을 cosine(또는 0에서 확정한 dot)으로 **명시**. 기본 L2 금지.
- 사이드카 `_index_header.json`을 **벡터 적재 완료 후 마지막에** 기록(os.replace 원자 교체). 헤더 정본 4필드 `embedding_model`(= `<bge_id>`), `dim`(=1024), `normalized`, `metric`을 기록한다(필드명 `model` 사용 금지 — 정본은 `embedding_model`).

**5) before/after 평가** — `eval/evaluate.py` 사용 + 비교 리포트 스크립트 `scripts/compare_embed.py` (신규)
`evaluate` 시그니처는 **불변**이다(인덱스 디렉토리/모델 인자 추가 금지). 따라서 `compare_embed.py`는 아래 두 방식 **중 하나**로 nomic/BGE-M3를 각각 측정한다. "인덱스 경로를 evaluate에 넘긴다"는 식으로 기술하지 않는다.
- **(a) settings 전환 방식**: `config/settings.yaml`의 활성 `embed_model`/`index_dir`/`prefix`를 `nomic`(→`data/index_nomic`)으로 설정해 `evaluate(...)`를 1회 호출(before), 이어 `bge`(→`data/index_bge`, `<bge_id>`)로 전환해 `evaluate(...)`를 1회 호출(after). 두 호출 모두 `use_bm25=False, use_rerank=False`(dense-only) 고정.
- **(b) search_fn 주입 방식**(정본 하네스 형태): nomic용·BGE-M3용 `search_fn`을 각각 구성(각 search_fn은 해당 모델로 질의 임베딩 + 해당 인덱스 디렉토리 사용)해 정본 하네스에 주입하고 `evaluate`를 각각 호출.
- 어느 방식이든 **질의 임베딩 모델·prefix·dim이 해당 인덱스(헤더)와 일치**해야 하며(불일치 시 조기 차단), before/after는 동일 골든셋·동일 옵션으로만 다르게 한다.
- 두 `EvalResult`의 `recall_at_1/3/5/10`, `mrr`, `ndcg_at_10`, 그리고 **회귀 케이스 per_query**를 나란히 출력.
- **채택 게이트**: `after.recall_at_1 >= before.recall_at_1` **그리고** 회귀 케이스("연차 휴가 신청 절차") per_query에서 정답이 1위(Recall@1=0→1)일 때만 "채택". 둘 중 하나라도 미달이면 "롤백"으로 판정.

**6) 채택/롤백 반영**
- 채택: `config/settings.yaml` 활성 임베딩을 BGE-M3(`<bge_id>`)로 확정, 활성 `index_dir`을 `data/index_bge/`로. 헤더 norm 재측정 결과(`normalized`) 반영. search/evaluate가 이 활성 설정을 따르는지 재확인.
- 롤백: `config/settings.yaml`을 nomic 프로파일로 되돌리고 활성 `index_dir`을 `data/index_nomic/`로 복구. BGE-M3 인덱스는 삭제하거나 보관만 하고 비활성.
- 어느 쪽이든 `before/after` 수치 리포트를 산출물로 남긴다.

### 완료 기준 (통과 못 하면 다음 페이즈 금지)
1. `scripts/measure_bge_norm.py` 실행 시 `dim=1024`가 나오고, norm 실측 결과로 `needs_normalize` 판정이 출력된다(`norm==0` 벡터는 정규화 제외로 별도 처리).
2. `embed(["연차 휴가 신청 절차"], model="<bge_id>")`가 1024차원 벡터를 반환하고, 정규화 정책 적용 후 norm 가드(`abs(norm-1.0)<1e-3`)를 통과한다(또는 정규화 분기로 통과).
3. `data/index_bge/`가 빌드되고 `_index_header.json`에 `embedding_model`(=`<bge_id>`), `dim=1024`, `metric`(cosine/dot 명시), `normalized`가 기록되어 있다(필드명 `model` 아님).
4. `scripts/compare_embed.py`가 before/after Recall@1·MRR·nDCG와 회귀 케이스 per_query를 출력하고, **"채택" 또는 "롤백" 중 하나의 판정**을 명확히 낸다. before/after 모두 질의 임베딩 모델·prefix·dim이 해당 인덱스와 일치한다.
5. 판정에 따라 `config/settings.yaml` 활성 프로파일과 활성 `index_dir`이 일관되게 설정되고(채택→bge, 롤백→nomic), search/evaluate가 이 활성 설정을 따른다. 다른 변수는 그대로다.

### 검증 명령
```bash
# 0) LMStudio에 BGE-M3 로드 확인 + 실제 id 기록 (이후 <bge_id> 자리에 이 값을 사용)
curl -s http://localhost:1234/v1/models | python -c "import sys,json; print([m['id'] for m in json.load(sys.stdin)['data']])"
# → 출력된 BGE-M3 id 문자열을 <bge_id>로 고정해 아래 모든 명령에 대입(하드코딩 금지)

# 1) norm 재측정 (dim=1024 / needs_normalize 판정 / norm==0 스킵)
python scripts/measure_bge_norm.py   # 내부에서 <bge_id> 사용

# 2) embedder 차원·norm 가드 확인 (회귀 케이스 질의로) — <bge_id>를 실제 id로 치환
python -c "from src.embed.embedder import embed; v=embed(['연차 휴가 신청 절차'], model='<bge_id>'); import math; n=math.sqrt(sum(x*x for x in v[0])); print('dim=',len(v[0]),'norm=',round(n,6))"

# 3) BGE-M3 전체 재인덱싱 (새 디렉토리)
#  3a) cli에 reindex가 있으면(사전조건 6 확인 결과) — 플래그명은 실제 cli.py에 맞게:
python -m src.app.cli reindex --embed-model '<bge_id>' --index-dir data/index_bge
#  3b) reindex 서브커맨드가 없으면 폴백: 인덱싱 모듈 직접 호출 빌드 스크립트 실행
#      (예) python scripts/build_index_bge.py --embed-model '<bge_id>' --index-dir data/index_bge
#  → 헤더 확인 (정본 필드명 embedding_model 사용, 'model' 아님)
python -c "import json; h=json.load(open('data/index_bge/_index_header.json')); print(h.get('dim'), h.get('metric'), h.get('embedding_model'), h.get('normalized'))"

# 4) before/after 비교 + 채택 게이트 (settings 전환 또는 search_fn 주입 — evaluate에 인덱스 경로 인자 안 넘김)
python scripts/compare_embed.py
# 기대 출력 예:
#   nomic   : R@1=0.xx R@3=.. R@5=.. R@10=.. MRR=.. nDCG@10=..  [연차휴가: rank=3]
#   bge-m3  : R@1=0.xx R@3=.. R@5=.. R@10=.. MRR=.. nDCG@10=..  [연차휴가: rank=1]
#   VERDICT : 채택 (Recall@1 개선 + 회귀케이스 0→1)   또는   롤백 (미개선)

# 5) 판정 반영 후 활성 설정 일관성 확인 (활성 embed_model·index_dir·prefix·metric)
grep -nE "embed|dim|metric|index_dir|prefix" config/settings.yaml
```
