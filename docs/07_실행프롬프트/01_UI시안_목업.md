# 01단계: UI 시안(목업) — Vite로 디자인 키트 3화면 폐쇄망 띄우기

### 이 페이즈의 목표
백엔드(환경셋업~리랭킹) 없이, 이미 완성된 React 디자인 키트 3화면(A 검색·B 인덱싱·C 평가)을 **Vite 프로젝트(`web/`)로 옮겨 폐쇄망 vendoring**하고, 가짜 응답(mock)으로 질의·인덱싱·평가 인터랙션이 실제로 도는 것(가짜 지연·로딩·상태 전이)을 `npm run dev`로 띄워 본다. **새로 그리지 않는다 — 키트를 그대로 살린다.**

### 사전 조건 (컨텍스트 초기화 후 단독 실행)
- 디자인 키트 정본 존재 (리포 루트 `docs-em/` 기준, **경로에 공백·한글 포함 → 셸에서 따옴표 필수**):
  - `디자인/docs-em Design System/styles.css` (진입: `tokens/*` import만)
  - `디자인/docs-em Design System/tokens/{colors,typography,spacing,base,fonts}.css`
  - `디자인/docs-em Design System/_ds_bundle.js` (~120KB, 전역 `window.DocsEmDesignSystem_afe3d1`로 21개 컴포넌트 노출. **이미 `React.createElement` 클래식 IIFE로 트랜스파일됨 → 런타임 Babel 불필요**)
  - `디자인/docs-em Design System/ui_kits/docs-em/{AppShell,QueryScreen,IndexScreen,EvalScreen}.jsx`
  - `디자인/docs-em Design System/assets/logo-mark.svg`
- Node ≥ 24, npm ≥ 11 설치됨(`node -v`로 확인). **백엔드·Python·서버 불필요** — 이 페이즈는 프론트만이며 어떤 백엔드 함수/서버도 호출하지 않는다.
- 작업 디렉토리는 리포 루트(`docs-em/`). 산출물은 신규 `web/` 폴더에만 만든다.
- **정본 키트는 읽기 전용 출발점**으로 취급하고 `web/`로 복사해 작업한다(정본 파일 수정 절대 금지). 편집은 `web/` 복사본에만, 그것도 아래 ③④⑧의 최소 변경만 허용한다.

### 키트 구조 실측 (이 페이즈의 전제 — 발명 금지)
- 화면 4 `.jsx`는 **ESM `export`가 없다.** 각 파일은 모듈 평가 시점에 `const { … } = window.DocsEmDesignSystem_afe3d1;`로 DS 전역을 구조분해하고, 끝에서 `Object.assign(window, { QueryScreen })`(AppShell은 `{ AppShell, Logo, SideRail, RailSection }`)로 자신을 전역에 노출한다. → **`_ds_bundle.js`가 먼저, 그다음 화면들이 순서대로 평가되어야** `undefined` 컴포넌트 에러가 안 난다.
- 키트는 전역 `React`/`ReactDOM`을 기대한다(`React.useState`, `ReactDOM.createRoot`). 정본 `index.html`은 이를 UMD CDN으로 제공했다 — 그 자리를 로컬 ESM + `window` 노출로 대체한다.
- 공통 헤더는 `AppShell.jsx`의 `TrustCluster()`(정본 기준 line 45 부근) 한 곳뿐이며, 여기에 `ConnectionBadge endpoint="localhost:1234"` · `ExternalCallsCounter count={0}` · `100% LOCAL`이 들어 있다. **3화면이 이 헤더를 공유**한다.

### 절대 규칙 (발췌 — 위반 시 미완)
- **외부 호출 0건(INV-3)**: 빌드 산출물·런타임 어디에도 `unpkg`/`jsdelivr`/`googleapis`/`gstatic`/`cdn` 등 외부 URL이 0건. 폰트(Pretendard·JetBrains Mono)·React·아이콘·번들 전부 로컬 vendoring. CDN `<script>`·`@import url(http…)` 전부 제거. (단, SVG의 `xmlns="http://www.w3.org/2000/svg"`는 네트워크 호출이 아니므로 허용 — 검증 grep에서 `w3.org`는 제외한다.)
- **신뢰 시그널 1급 고정**: `localhost:1234` · `EXTERNAL CALLS: 0` · `100% LOCAL`을 3화면 공통 헤더(`AppShell.TrustCluster`)에 항상 노출. `ExternalCallsCounter count={0}` 유지(mock은 외부 호출을 안 하므로 0이 사실).
- **정직 수치**: Recall@1=0 · 유사도 격차 −0.018 · REGRESS를 미화하지 않는다. `prompt_tokens=0`엔 `(unreliable)` 앰버.
- **BGE-M3(1024) 미채택 DECISION**: 화면 A에 노출 금지. 화면 B 좌레일에서만 `⚠` + DECISION 모달로.
- **(mock) 정직 표기**: 모든 데이터는 가짜다. 화면이 스스로 "가짜다"를 말하게 한다. `+rrf` 회귀 0→1 전환·안정성 게이지 등 SSOT 실측 근거 없는 시각화는 `(데모)` 배지.
- **디자인 토큰·DS 컴포넌트만**: `var(--…)` 토큰과 `_ds_bundle.js` 컴포넌트만 사용. 새 HEX·새 컴포넌트 금지. 컬러 이모지(픽토그램) 금지. 상태 마커 `⚠/✔/✓/⟳/○` 기호 글리프는 허용.
- **가짜값 vs 사실값**: 데모 식별자(`hr-0412`/`tr-0203`·`p.N`)는 가짜. 확정 사실값(768·norm1·−0.018·0.687→0.695·12,972자·ctx 8192/65536·`localhost:1234`)만 진짜. 데모 문서는 `휴가규정.pdf`·`출장경비.docx` 2종만.

### 작업 지시 (순서대로)

**① `web/` Vite 프로젝트 생성**
- 리포 루트에 `web/` 생성. `web/package.json`에 deps `react@18.3.1`, `react-dom@18.3.1`, devDeps `vite`, `@vitejs/plugin-react`를 두고 `npm install`(로컬 npm 레지스트리/캐시로 설치 — 이 설치 자체는 빌드타임이며 INV-3는 런타임·산출물 기준).
- `scripts`: `"dev": "vite"`, `"build": "vite build"`, `"preview": "vite preview"`.
- `web/vite.config.js`: `@vitejs/plugin-react`만. `base: './'`(상대 경로)로 두어 어떤 정적 서버에도 서빙되게. **별도 esbuild 스크립트·`public/` 스크립트 로드 전략은 쓰지 않는다**(아래 ④ Option B 단일 채택).

**② 디자인 키트 복사 (정본 → `web/`)**
- `styles.css` → `web/src/ds/styles.css`
- `tokens/` 전체 → `web/src/ds/tokens/`
- `_ds_bundle.js` → `web/src/ds/_ds_bundle.js`
- `ui_kits/docs-em/{AppShell,QueryScreen,IndexScreen,EvalScreen}.jsx` → `web/src/screens/`
- `assets/logo-mark.svg` → `web/src/ds/assets/`
- 복사본 내용은 ③④⑧의 최소 변경 외에는 수정하지 않는다.

**③ 폰트·React vendoring + CDN 제거 (INV-3 핵심)**
- **폰트 바이너리** (`web/src/ds/assets/fonts/`에 로컬로):
  - `Pretendard-{Regular,Medium,SemiBold,Bold}.woff2` 4개
  - `JetBrainsMono-{Regular,Medium,SemiBold,Bold}.woff2` 4개 (가중치 400/500/600/700)
  - **정본 리포에는 woff2 바이너리가 아직 없다.** 사용자가 이미 제공했으면 그 경로에서 복사. **없으면 사용자에게 1회 요청**: "다음 8개 woff2를 `web/src/ds/assets/fonts/`에 넣어주세요 — Pretendard 4개(Regular/Medium/SemiBold/Bold) + JetBrains Mono 4개(400/500/600/700)." (JetBrains Mono도 Pretendard와 **동급**으로 명시 — 정본 fonts.css엔 JetBrains가 `@import`로만 있어 woff2가 통째로 빠져 있으므로 4개 전부 받아야 한다.)
- **`web/src/ds/tokens/fonts.css` 재작성**:
  - `@import url("https://fonts.googleapis.com/…")` 라인 **삭제**(JetBrains Mono용 `@import` 제거).
  - Pretendard 4개 `@font-face`의 `src`를 `url("../assets/fonts/Pretendard-Regular.woff2") format("woff2")` 식 **로컬 상대경로**로 재지정(파일명만 가중치별로 교체).
  - JetBrains Mono **`@font-face` 4블록을 신규 작성**(정본엔 없으므로 새로 만든다). 예:
    ```css
    @font-face {
      font-family: "JetBrains Mono";
      font-weight: 400; font-style: normal; font-display: swap;
      src: url("../assets/fonts/JetBrainsMono-Regular.woff2") format("woff2");
    }
    /* 500/600/700 동일 패턴으로 Medium/SemiBold/Bold 3블록 추가 */
    ```
  - **woff2 부재 시 폴백 정직 기술**: `font-family` 체인의 폴백은 Inter/IBM Plex Mono → `system-ui`/`monospace`다. Inter·IBM Plex Mono도 vendoring되지 않았으므로, woff2가 없으면 실제로는 **`system-ui`/`monospace`로만 렌더**된다. **화면이 깨지지는 않으나 디자인 의도와 차이가 난다** — 이 사실을 사용자에게 함께 고지한다.
- **React**: CDN `<script>`(`unpkg react/react-dom/@babel/standalone`) 전부 제거. React는 로컬 `node_modules`에서 ESM import. **별도 모듈로 전역 노출**(아래 ④ 호이스팅 함정 회피):
  - `web/src/ds/expose-react.js` 신규 작성:
    ```js
    import React from "react";
    import ReactDOM from "react-dom/client";
    window.React = React;
    window.ReactDOM = ReactDOM;
    ```
  - 이 파일을 엔트리에서 **가장 먼저** import한다(④ 참조).

**④ 키트의 `window` 전역 패턴을 Vite에 맞게 로드 — Option B 단일 채택 (원본 최대 보존)**
- 화면 4 `.jsx`는 `export`가 없는 **side-effect 모듈**이고 로드 순서에 의존한다. `@vitejs/plugin-react`가 이 `.jsx`들을 JSX 변환하며, side-effect import는 tree-shake되지 않고 평가된다(`Object.assign(window, …)`가 그대로 실행됨). **Babel standalone·esbuild 사전 트랜스파일·`public/` 스크립트 로드는 쓰지 않는다.**
- **호이스팅 함정 주의(반드시 준수)**: ESM `import`는 호이스팅되므로, 엔트리 본문에 `window.React = React`를 직접 쓰면 그 대입이 import된 `_ds_bundle.js`·화면 모듈 평가보다 **늦게** 실행된다. 그래서 React 전역 노출을 ③의 `expose-react.js`로 **분리**하고, 엔트리 **최상단 첫 import**로 둔다. 그러면 import 평가 순서상 React 전역이 번들·화면보다 먼저 평가된다.
- `web/src/main.jsx` (엔트리) — **import 순서가 곧 로드 순서**:
  ```js
  import "./ds/expose-react.js";        // 1) window.React / window.ReactDOM 먼저
  import "./ds/styles.css";             // 2) 토큰·폰트 (styles.css가 tokens/* import)
  import "./ds/_ds_bundle.js";          // 3) window.DocsEmDesignSystem_afe3d1 채움
  import "./screens/AppShell.jsx";      // 4) window.AppShell/Logo/SideRail/RailSection
  import "./screens/QueryScreen.jsx";   // 5) window.QueryScreen
  import "./screens/IndexScreen.jsx";   // 6) window.IndexScreen
  import "./screens/EvalScreen.jsx";    // 7) window.EvalScreen

  function App() {
    const [active, setActive] = window.React.useState("query");
    const Screen = active === "query" ? window.QueryScreen
                 : active === "index" ? window.IndexScreen
                 : window.EvalScreen;
    return window.React.createElement(
      window.AppShell,
      { active, onNavigate: setActive },
      window.React.createElement(Screen)
    );
  }
  window.ReactDOM.createRoot(document.getElementById("app"))
    .render(window.React.createElement(App));
  ```
  (정본 `index.html`의 `App()` 토글 로직과 동일하게 — `active`로 `window.QueryScreen / IndexScreen / EvalScreen`을 고르고 `window.AppShell`로 감싼다.)
- `web/index.html`: `<div id="app"></div>` + `<script type="module" src="/src/main.jsx"></script>`만. **CDN `<script>` 0건.**

**⑤ mock 모듈 작성 (`web/src/mock/`) — 실 API와 동일 시그니처**
- 나중 실 배선이 `web/src/api/client.js`로 교체만 하면 되도록 **동일 export 이름·동일 반환 모양**으로 작성. 모든 함수는 `setTimeout` 기반 가짜 지연(검색 300–700ms, 인덱싱은 단계별 진행 콜백). **`fetch`·서버 호출 없음.**
  - `mockAnswer({query, mode, model, topN}) → { text, citations:[{id,docId,page}], topk:[{rank,docId,page,score,isAnswer}], meta:{prompt_tokens:0 /*unreliable*/, retries:1, mode} }` — 데모: 정답 `휴가규정 hr-0412` 3위, 격차 −0.018.
  - `mockSearch({query, mode}) → topk[]` (벡터/BM25/하이브리드RRF Segmented 대응).
  - `mockEval({experiment}) → { recall_at_1, recall_at_3, recall_at_5, recall_at_10, mrr, ndcg_at_10, per_query:[…] }` — baseline/+bm25는 회귀(Recall@1=0), `+rrf`는 `(데모)` 성공.
  - `mockIndexUpload(files)`, `mockIndexRun(onStep)`(5노드 파이프라인 단계별 진행: pymupdf✔12 / kss✔1,842 / 청킹✔312 / 임베딩⟳188→312 / 저장○0), `mockIndexDocs() → docs[]`(손상 스캔 파싱 실패 행 포함).
- **mock 데이터의 가짜값/사실값은 절대 규칙을 따른다**(데모 식별자만 가짜, 확정 수치만 사실).

**⑥ 3화면 라우팅**
- 별도 라우터 라이브러리 없이 정본 그대로 `AppShell`의 `active`/`onNavigate` 상태 토글(검색/인덱싱/평가). 키트의 네비·헤더 그대로 사용(④의 `App()`이 이미 토글).

**⑦ 가짜 지연·로딩·상태로 인터랙션 동작**
- 화면 A: 질의 입력 후 검색 시 **로딩 상태 → mock 지연 → Top-k·인라인 각주[¹][²]·재시도1 메타 아코디언** 표시. 검색모드 Segmented(벡터/BM25/하이브리드RRF) 전환 시 `mockSearch` 재호출. cutoff 앰버 점선 유지.
- 화면 B: 드롭존에 파일 올리면 거부 목록/색인 테이블 갱신, "재색인" 시 **파이프라인 5노드가 단계별 진행(⟳ → ✔)** 되는 가짜 진행. 좌레일 BGE-M3 `⚠` 클릭 → DECISION 모달.
- 화면 C: 실험 드롭다운(baseline/+bm25/+rrf) 변경 시 `mockEval` 재호출 → 거대 Recall@1 KPI·before▸after 테이블 갱신. `+rrf` 선택 시 0▸1.0 성공 데모(red→green 플래시)는 `(데모)` 배지와 함께.
- 모든 상태 전이는 가짜 지연을 거치되, 백엔드 호출은 없다(전부 `web/src/mock/`).
- **인터랙션 배선 위치**: mock 호출은 화면 `.jsx` 본문에서 일어나야 한다. 정본 화면이 이미 자체 mock 상수를 들고 있다면 **그 상수를 `web/src/mock/`의 비동기 함수 호출로 바꾸는 최소 편집**만 복사본에 가한다(레이아웃·DS 컴포넌트·수치는 보존). 새 화면을 그리지 않는다.

**⑧ `(mock)` 표식 — 공통 헤더 1회 (화면 본문 보존)**
- **1순위(권장)**: `web/src/screens/AppShell.jsx`의 `TrustCluster()` 한 곳에 DS `Tag`(또는 `StatusPill`)로 `(mock)` 배지를 **1회** 추가한다. AppShell이 3화면 공통 헤더이므로 이 한 번으로 3화면 모두에 노출되고, **QueryScreen/IndexScreen/EvalScreen 본문은 손대지 않는다**(정본 보존). `Tag`는 이미 `TrustCluster`가 import하는 컴포넌트라 새 의존 불필요.
- **2순위(옵션)**: 화면별로 더 눈에 띄게 두고 싶으면 각 스크린 상단에 추가 가능하나, 그만큼 화면 `.jsx` 편집이 늘어난다(보존성 ↓). 기본은 1순위만 한다.
- SSOT 실측 근거 없는 시각화(`+rrf` 성공·안정성 게이지)엔 EvalScreen에 이미 있는 `(데모)` 마커를 유지/사용한다(신규 작성 시에도 `(데모)`로).

1. `cd web && npm run dev` → 브라우저에서 3화면(검색/인덱싱/평가)이 뜨고 네비 토글이 동작(전역 `undefined` 컴포넌트 에러 없음 = 로드 순서 보장 확인).

참고 사실(정본 실측 — 이 페이즈에서 직접 확인):
- 키트 정본 위치는 `디자인/docs-em Design System/ui_kits/docs-em/` (리포 루트 직하가 아님 — 디자인 시스템 폴더 내부).
- DS 전역 네임스페이스 `window.DocsEmDesignSystem_afe3d1` (`_ds_bundle.js` ~120KB). 화면 4 `.jsx`는 **`export` 없음** — 모듈 평가 시 이 전역을 구조분해하고 `Object.assign(window, …)`로 자신을 노출 → **로드 순서 의존**. `_ds_bundle.js`는 이미 `React.createElement` 클래식 IIFE라 런타임 Babel 불필요.
- 외부 URL은 현재 `tokens/fonts.css`(Pretendard ×4 jsDelivr `@font-face` + JetBrains Mono Google Fonts `@import`, **JetBrains는 `@font-face` 없음**)와 `ui_kits/docs-em/index.html`(React/ReactDOM UMD + `@babel/standalone` unpkg) 두 곳뿐. SVG `xmlns="http://www.w3.org/2000/svg"`는 네트워크 호출 아님.
- **폰트 woff2 바이너리는 리포에 아직 없음** → ③에서 사용자에게 8개(Pretendard 4 + JetBrains Mono 4) 1회 요청 필요. 부재 시 폴백은 Inter/IBM Plex Mono도 미vendoring이라 실제로는 `system-ui`/`monospace`로 렌더(깨짐은 없으나 디자인 차이 있음).
