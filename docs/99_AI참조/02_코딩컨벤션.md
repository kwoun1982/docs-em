## 1. 언어 규칙 — 식별자 영어, 설명 한국어

| 대상 | 언어 | 예시 |
|------|------|------|
| 변수·함수·클래스·모듈명 | 영어 (snake_case / PascalCase) | `chunk_text`, `EmbeddingClient` |
| 상수 | 영어 (UPPER_SNAKE_CASE) | `EMBEDDING_DIM = 768` |
| 주석 (`#`) | 한국어 | `# 출력이 이미 L2 정규화됨 → 재정규화 금지` |
| docstring | 한국어 | `"""질의를 768차원 벡터로 임베딩합니다."""` |
| 로그 메시지 | 한국어 | `logger.info("골든셋 %d건 평가 시작", n)` |
| 커밋 메시지 | 한국어 (제목 영어 동사 허용) | `fix: reasoning_content 폴백 누락 수정` |

- 식별자에 한글·로마자 혼용 금지. (`연차_chunk` 금지)
- 도메인 용어(연차, 출장경비 등)는 데이터 값으로만 등장하고, 식별자에는 영어 표기(`annual_leave`)를 씁니다.

## 2. 포맷·린트 — ruff 단일 도구

포맷터·린터는 **ruff**로 통일합니다(black/isort/flake8 중복 도입 금지).

```toml
# pyproject.toml
[tool.ruff]
target-version = "py311"
line-length = 100

[tool.ruff.lint]
select = ["E", "F", "I", "UP", "B", "SIM", "RUF"]
# E/F: pycodestyle·pyflakes, I: isort, UP: pyupgrade,
# B: bugbear, SIM: simplify, RUF: ruff 고유 규칙

[tool.ruff.format]
quote-style = "double"
```

- 들여쓰기 4 스페이스, 줄길이 100, 임포트는 ruff(`I`)가 자동 정렬합니다.
- 커밋 전 `ruff format . && ruff check --fix .`를 실행합니다.
- 룰셋 세부(`line-length`·`select` 등)는 **DECISION**. 위 값은 권장 기본값입니다.

## 3. 타입힌트 — 전부 명시

- 모든 공개 함수·메서드의 인자·반환에 타입힌트를 답니다. `Any`는 최후의 수단이며 이유를 주석으로 남깁니다.
- Python 3.11+ 문법을 씁니다: `list[str]`, `dict[str, float]`, `X | None`(typing.Optional 대신).
- 데이터 구조는 `@dataclass` 또는 `pydantic` 모델로 정의합니다. 어느 쪽을 표준으로 할지는 **DECISION**. (함수 시그니처에 쓰는 인메모리 타입 `Chunk`·`SearchHit`·`EvalResult`의 정본 정의는 → [02_내부API_인터페이스.md](../04_아키텍처_API/02_내부API_인터페이스.md) §1 참조.)

```python
from dataclasses import dataclass

@dataclass
class SearchHit:
    doc_id: str
    score: float
    text: str

def embed_query(text: str) -> list[float]:
    """질의를 768차원 벡터로 임베딩합니다. 출력은 이미 L2 정규화되어 있습니다."""
    ...
```

- `EMBEDDING_DIM = 768`은 상수로 한 곳에서만 정의하고 매직넘버를 코드에 흩지 않습니다. 차원·prefix·norm 등 임베딩 실측 근거는 → [시작프롬프트.md](../../시작프롬프트.md) §B-12(+ §B-2) 참조.

## 4. 에러처리 패턴

원칙: **조용한 실패 금지, 광범위 except 금지, 실패는 명시적으로.** (내부 API 전반의 폴백 계약은 → [02_내부API_인터페이스.md](../04_아키텍처_API/02_내부API_인터페이스.md) §7 참조.)

```python
# 나쁨 — 무엇이 터졌는지 삼킴
try:
    resp = client.embeddings.create(...)
except Exception:
    return []

# 좋음 — 잡을 예외를 좁히고, 맥락을 붙여 다시 던짐
try:
    resp = client.embeddings.create(model=EMBED_MODEL, input=texts)
except (APIConnectionError, APITimeoutError) as e:
    raise EmbeddingError(f"LMStudio 임베딩 호출 실패: {texts[0][:30]!r} 외 {len(texts)-1}건") from e
```

규칙:

- `except Exception:`로 모든 것을 삼키지 않습니다. 잡는 예외 타입을 좁힙니다.
- 예외를 다시 던질 때 `raise ... from e`로 원인 체인을 보존합니다.
- 도메인 예외(`EmbeddingError`, `GenerationError`, `RetrievalError`)를 정의해 호출부가 분기할 수 있게 합니다.
- 빈 결과(`[]`)와 실패(예외)를 구분합니다. "검색 결과 0건"과 "검색 호출 실패"는 다른 사건입니다.
- LMStudio SDK 예외(미로드·네트워크)는 래퍼가 **삼키지 않고 그대로 전파**합니다. 호출자가 재시도·폴백 모델 전환을 결정합니다(→ [02_내부API_인터페이스.md](../04_아키텍처_API/02_내부API_인터페이스.md) §2.1).
- 단, 생성모델 `finish_reason=length` 재시도는 래퍼 내부에서 처리하고 호출부에 새지 않게 합니다(§5, INV-6).

## 5. LMStudio 호출 — 반드시 래퍼 경유 (핵심 규칙)

> 이것이 본 프로젝트에서 가장 자주 깨지는 규칙입니다. 제약의 실측 근거는 → [시작프롬프트.md](../../시작프롬프트.md) §B-4(reasoning Blocking)·§B-7(래퍼 코드)·§B-12(실측 요약)를, 함수 계약은 → [02_내부API_인터페이스.md](../04_아키텍처_API/02_내부API_인터페이스.md) §2를 참조하세요.

### 5.1 절대 금지 (INV-3)

- 외부 클라우드 API(OpenAI/Anthropic/Cohere 등) 호출 코드를 작성하지 않습니다. 모든 LLM·임베딩은 **로컬 LMStudio**(`http://localhost:1234/v1`, OpenAI 호환, `api_key`는 더미)만 사용합니다. OpenAI SDK는 쓰되 `base_url`만 LMStudio로 교체합니다.
- 비즈니스 로직·노트북·스크립트에서 `openai` SDK의 `client.chat.completions.create(...)` / `client.embeddings.create(...)`를 **직접** 호출하지 않습니다. 반드시 프로젝트 래퍼(`chat()`, `embed()`)를 경유합니다. reasoning·norm·입력 가드 같은 LMStudio 특수 처리는 **래퍼 한 곳에만 가둡니다**(호출부 분산 금지).

### 5.2 생성 래퍼가 책임지는 것 (INV-2 / INV-5 / INV-6)

생성 호출 래퍼는 다음 세 가지를 **반드시** 처리합니다(미처리 시 결과가 빈 문자열이거나 잘립니다). 세 생성 모델 모두에서 재현된 실측입니다 — `qwen/qwen3.6-35b-a3b`, `qwen/qwen3.5-9b`, `google/gemma-4-e4b`(9B/4B 파라미터 수는 모델명 유추라 `[추정]`, ctx 수치는 실측).

| 제약 (실측) | 래퍼의 처리 | 대응 불변식 |
|------|------|------|
| 세 모델 모두 `message.content`가 빈 문자열로 옴 | `content`가 비면 `message.reasoning_content`로 폴백 | INV-2 |
| 답에 `<think>...</think>`가 섞임 | 폴백 텍스트에서 `<think>` 블록 제거 + 닫는 태그만 남은 경우 마지막 `</think>` 이후를 최종답으로 추출 | INV-5 |
| 무한사고로 `finish_reason=length` 잘림 (한 문장 질문에 12,972자) | `length`면 `max_tokens`를 늘려 1회 재시도, 2회째도 `length`면 경고 부착(빈값 방치 금지) | INV-6 |
| `/no_think`·`enable_thinking=false`·`reasoning_effort=low`·`max_tokens=4000` 전부 무효 | 위 파라미터에 의존하지 않음 (먹히지 않음을 전제로 설계) | INV-2 |

```python
import re

def _clean_reasoning(text: str) -> str:
    """reasoning_content 폴백 시 사고과정 정제. <think> 블록 제거 후 최종답 추출."""
    # <think>...</think> 블록 제거 (모델별 구분자에 맞게 조정)
    text = re.sub(r"<think>.*?</think>", "", text, flags=re.DOTALL).strip()
    # 닫는 태그만 있는 경우(여는 태그 없이 잘림): 마지막 </think> 이후를 최종답으로
    if "</think>" in text:
        text = text.split("</think>")[-1].strip()
    return text

def generate(self, messages: list[dict], *, model: str = "google/gemma-4-e4b",
             max_tokens: int = 4000, _retry: bool = True) -> str:
    """생성 호출. content 빈값이면 reasoning_content로 폴백하고,
    finish_reason=length면 max_tokens를 늘려 1회 재시도합니다.
    근거: 시작프롬프트.md §B-4·§B-7 / 불변식 INV-2·INV-5·INV-6."""
    resp = self._call(messages, model=model, max_tokens=max_tokens)
    choice = resp.choices[0]
    msg = choice.message
    content = (msg.content or "").strip()
    if not content:
        # 실측: 세 모델 모두 content 빈값, 실제 답이 reasoning_content에 옴
        content = _clean_reasoning((getattr(msg, "reasoning_content", "") or "").strip())
    if choice.finish_reason == "length" and _retry:
        # 실측: 무한 사고로 length 잘림(12,972자) → max_tokens 상향 1회 재시도.
        # 텍스트 유무와 무관하게 재시도(INV-6: 잘린 채 방치 금지).
        return self.generate(messages, model=model, max_tokens=max_tokens * 3, _retry=False)
    if choice.finish_reason == "length":
        content += "\n\n[경고] 응답이 잘렸습니다(finish_reason=length). LMStudio 프리셋의 reasoning 종료 토큰을 점검하세요(§B-4)."
    return content
```

> 위 코드는 [시작프롬프트.md](../../시작프롬프트.md) §B-7 정본 패턴 및 [02_내부API_인터페이스.md](../04_아키텍처_API/02_내부API_인터페이스.md) §2.1 계약과 일치시킨 예시입니다. 재시도는 `finish_reason=length`이면 텍스트 유무와 무관하게 1회 발동합니다(폴백으로 일부 텍스트가 잡힌 잘림 케이스도 재시도). 재시도 증가폭(`max_tokens*3`)·횟수(1회)는 형제 계약(§2.1·§7)에 확정된 값이며, 변경 시 **DECISION**으로 두 문서를 함께 갱신합니다. 폴백 게이트의 도입·검증 시점은 로드맵 L2입니다(→ [시작프롬프트.md](../../시작프롬프트.md) §B-10).

### 5.3 임베딩 래퍼가 책임지는 것 (INV-1 / INV-4)

| 제약 (실측) | 래퍼의 처리 | 대응 불변식 |
|------|------|------|
| 출력이 이미 L2 정규화됨 (norm=1.0) | **재정규화 금지.** 받은 벡터를 그대로 사용 | INV-1 |
| 모델 교체 시 정규화 깨질 수 있음 | `abs(norm - 1.0) < 1e-3`를 assert (norm 가드) | INV-1 |
| 차원 768 고정 | `EMBEDDING_DIM` 상수와 일치 검증(assert) | INV-1 |
| `usage.prompt_tokens=0` 오보고 | 토큰 회계에 LMStudio usage를 신뢰하지 않음 | INV-4 |
| 입력 길이 한계 `[확인필요]` | 임베딩 입력 가드를 호출 전에 적용해 무음 잘림 차단 | INV-4 |

```python
import math

def embed(self, texts: list[str]) -> list[list[float]]:
    """배치 임베딩. 출력은 이미 정규화되어 있으므로 재정규화하지 않습니다.
    입력 순서와 1:1 대응. 근거: 시작프롬프트.md §B-7·§B-12 / INV-1·INV-4."""
    resp = self._client.embeddings.create(model=self._model, input=texts)
    vecs = [d.embedding for d in resp.data]
    for vec in vecs:
        assert len(vec) == EMBEDDING_DIM, f"차원 불일치: {len(vec)}"  # 768 검증
    # norm 가드: 모델 교체(nomic→BGE-M3)로 정규화가 깨지면 여기서 즉시 감지
    n = math.sqrt(sum(x * x for x in vecs[0]))
    assert abs(n - 1.0) < 1e-3, (
        f"임베딩 norm={n:.4f} (1.0 아님). 모델 교체로 정규화가 깨졌을 수 있습니다. "
        f"metric/정규화 분기를 재검토하세요(A-2 ①)."
    )
    return vecs  # 재정규화하지 않음
```

- 입력은 **배치(`list[str]`)** 시그니처를 표준으로 합니다(단일 문자열은 `[text]`로 감싸 호출). 인덱싱 성능에 유리하고 형제 계약(§2.2)과 일치합니다.
- 임베딩 입력 최대 길이(nomic-v1.5 입력 한계)는 **[확인필요]**입니다 — Phase 0 환경 점검에서 실측합니다. `prompt_tokens=0` 오보고로 초과 입력이 **무음 잘림**되므로, 입력 가드(문자 수 1차, 토큰 필요 시 HF 토크나이저 로컬 계산)를 호출 전에 둡니다(→ [02_내부API_인터페이스.md](../04_아키텍처_API/02_내부API_인터페이스.md) §3.2, 불변식 INV-4).
- 모델 교체(nomic→BGE-M3) 시 차원이 768→1024로 바뀌고 norm 보장이 깨질 수 있습니다. 차원은 설정에서 읽고 하드코딩하지 않으며, norm 가드가 트립하면 정규화 분기를 추가하고 인덱스를 재생성합니다(BGE-M3 도입은 **DECISION** — → [시작프롬프트.md](../../시작프롬프트.md) §B-5).

## 6. 비밀값·설정 — .env

- 엔드포인트·모델명·경로 등 환경 의존 값은 `.env`로 분리하고, 코드에 하드코딩하지 않습니다.
- `.env`는 커밋하지 않습니다(`.gitignore`에 등록). 대신 `.env.example`을 커밋해 키 목록을 공유합니다.
- LMStudio의 `api_key`는 더미지만, 동일 패턴으로 `.env`에서 읽어 외부 API 키 누출 습관을 들이지 않습니다.

```bash
# .env.example  (값 없이 키 목록만)
LMSTUDIO_BASE_URL=http://localhost:1234/v1
LMSTUDIO_API_KEY=lm-studio          # 더미값, 외부 유출 무관
EMBED_MODEL=text-embedding-nomic-embed-text-v1.5
# 답변 생성 기본 모델 — DECISION (시작프롬프트.md §B-9 #8: gemma-4-e4b, ctx 65536)
GEN_MODEL=google/gemma-4-e4b
```

- **DECISION**: 답변 생성 기본 모델은 `google/gemma-4-e4b`(긴 컨텍스트 ctx 65536)로 잠정 결정되어 형제 문서([02_내부API_인터페이스.md](../04_아키텍처_API/02_내부API_인터페이스.md) §2.1)의 `chat()` 기본값과 일치합니다. 4B급이라 추론 품질 한계가 있을 수 있어 §B-4 해결 후 재평가합니다(→ [시작프롬프트.md](../../시작프롬프트.md) §B-9 #8). 모델명을 코드/예시에 박을 때 9B(`qwen3.5-9b`)·4B(`gemma-4-e4b`) 파라미터 수는 모델명 유추라 `[추정]`입니다.
- 설정은 한 곳(`config.py` 또는 pydantic `Settings`)에서 로드하고, 모듈 곳곳에서 `os.environ`을 직접 읽지 않습니다.

## 7. 테스트 규칙

| 구분 | 대상 | 외부 의존 |
|------|------|----------|
| 단위 테스트 | 청킹·`<think>` 정제·RRF 결합·차원/norm 검증 등 순수 로직 | LMStudio 호출 **모킹** |
| 통합 테스트 | 래퍼가 실제 LMStudio에 붙는지 | `localhost:1234` 필요, 마커로 분리 |
| 평가(별개) | Recall@k/MRR/nDCG, 골든셋 회귀 | 평가 하네스가 담당 → [시작프롬프트.md](../../시작프롬프트.md) §B-8 |

규칙:

- **테스트는 외부 클라우드 API를 절대 호출하지 않습니다.** LMStudio 호출은 단위 테스트에서 모킹합니다(INV-3).
- LMStudio가 필요한 통합 테스트는 `@pytest.mark.integration`으로 분리해, 서버가 없으면 스킵할 수 있게 합니다.
- **연차 휴가 회귀 케이스는 반드시 골든셋에 포함합니다(INV-7).** "연차 휴가 신청 절차" 질의에서 정답(휴가규정)이 오답(출장경비)보다 위로 오는지 검증합니다. 현재 실측은 정답 3위, 격차 -0.018, Recall@1=0이므로 이 케이스는 **개선 목표이자 회귀 가드**입니다(근거: [시작프롬프트.md](../../시작프롬프트.md) §A-2 ④·§B-8, [02_내부API_인터페이스.md](../04_아키텍처_API/02_내부API_인터페이스.md) §6).
- 평가(품질 측정)와 테스트(동작 보장)는 다릅니다. 변수는 한 번에 하나씩 바꿔 측정합니다(INV-7, → [시작프롬프트.md](../../시작프롬프트.md) §B-8).
- 테스트 러너·커버리지 임계치는 **DECISION**. `pytest` 권장.

## 8. 빠른 체크리스트 (코드 작성·리뷰 시)

- [ ] 식별자 영어, 주석·docstring 한국어인가
- [ ] `ruff format` + `ruff check`를 통과하는가
- [ ] 공개 함수에 타입힌트가 있는가
- [ ] `except Exception:`으로 예외를 삼키지 않는가, `raise ... from e`인가
- [ ] LLM·임베딩 호출이 래퍼(`chat()`/`embed()`) 경유인가 (직접 `create()` 금지)
- [ ] 생성 래퍼가 `reasoning_content` 폴백 + `<think>` 정제 + `length` 1회 재시도(`max_tokens*3`)를 하는가 (INV-2/5/6)
- [ ] 임베딩 벡터를 **재정규화하지 않았는가** (이미 norm=1.0), norm 가드·차원 검증이 있는가 (INV-1)
- [ ] 임베딩 입력 가드가 호출 전에 있는가 (`prompt_tokens=0` 무음 잘림 차단, 한계값 [확인필요]) (INV-4)
- [ ] 외부 클라우드 API(OpenAI/Anthropic/Cohere) 호출이 없는가 (INV-3)
- [ ] 비밀값·엔드포인트가 `.env`에 있고 코드에 하드코딩되지 않았는가
- [ ] 테스트가 외부 API를 호출하지 않고 LMStudio를 모킹하는가
- [ ] 골든셋에 연차 휴가 회귀 케이스가 있는가 (INV-7)
