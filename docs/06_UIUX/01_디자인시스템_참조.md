# docs-em 디자인 시스템 — 개발 참조

> Claude Design이 [00_ClaudeDesign_프롬프트.md](./00_ClaudeDesign_프롬프트.md)를 바탕으로 생성한 **실제 디자인 시스템**(CSS 토큰 + 21개 React 컴포넌트 + 3화면 UI 키트)을 개발에서 참조하기 쉽게 정리한 문서입니다.
> **원본 자산 위치**: `디자인/docs-em Design System/` (저장소 루트 기준). docs 폴더 기준 상대경로는 `../../디자인/docs-em Design System/`.

## 2. 타이포그래피 (정본: `tokens/typography.css`)

- **UI 산세리프**: `--font-sans` = `Pretendard, Inter, …` — 한국어 본문·라벨·버튼. line-height `--leading-body`(1.6).
- **데이터 모노**: `--font-mono` = `JetBrains Mono, IBM Plex Mono, …` — 점수·차원·chunk_id·모델명·`localhost:1234`·`finish_reason` 등 **모든 데이터**. `.mono` 클래스가 `tabular-nums` 적용.
- **장문 한국어를 모노로 쓰지 말 것.**

| 토큰 | px | 용도 |
|---|---|---|
| `--text-xs` / `--text-sm` | 12 / 13 | 데이터·라벨 모노 |
| `--text-base` | 14 | 본문 |
| `--text-lg` | 16 | 섹션 제목 |
| `--text-2xl` / `--text-3xl` | 21 / 24 | 화면 제목 |
| `--text-4xl` | 30 | 1급 지표(Recall@1) 모노 단독 |
| `--text-6xl` | 56 | 거대 KPI(화면 C Recall@1) |

웨이트 `--weight-regular/medium/semibold/bold`(400/500/600/700). 트래킹: `--tracking-wider`(0.08em)는 대문자 eyebrow 라벨(`EXTERNAL CALLS: 0`, `100% LOCAL`)용.

유틸 클래스(`tokens/base.css`): `.mono`(데이터), `.eyebrow`(대문자 라벨).

## 3. 간격·모서리·모션 (정본: `tokens/spacing.css`)

- **간격**: 4px 단위. `--space-1`(4) ~ `--space-20`(80).
- **모서리**: `--radius-md`(6px) 기본 컨트롤/카드, `--radius-lg`(8px) 카드, `--radius-full` 알약.
- **그림자**: 다크 UI라 절제 — 기본은 hairline 보더, 그림자는 떠 있는 레이어(모달 `--shadow-overlay`)만.
- **모션**: `--dur-fast/base/slow`(120/180/280ms), `--ease-out`. **바운스·패럴랙스 금지.** `prefers-reduced-motion` 시 최종 상태만.
- **레이아웃 상수**: `--rail-width`(248px), `--topbar-height`(52px), `--content-max`(1200px).

## 4. 컴포넌트 카탈로그 (21개)

모두 React(JSX). `<Name>.prompt.md`에 사용법, `.d.ts`에 props 타입. 번들 로드 시 `window.DocsEmDesignSystem_afe3d1`로 접근.

### forms (입력/액션)

| 컴포넌트 | 용도 | 핵심 props·예시 |
|---|---|---|
| **Button** | 뷰당 핵심 액션 1개는 teal `primary` | `variant=primary\|secondary\|ghost\|danger`, `size=sm(26)\|md(32)\|lg(38)`, `iconLeft/Right` |
| **IconButton** | 툴바/인라인 아이콘 버튼 | `icon`, `variant=ghost\|outline`, `active` |
| **Input** | inset 트로프 + teal 포커스 | `icon`, `mono`(쿼리/경로/ID), `invalid`(레드 보더) |
| **Select** | 네이티브 select + chevron | `options=[string \| {value,label}]` |
| **Switch** | 이진 토글(on=teal) | `checked`, `onChange`, `label` |
| **Segmented** | 배타 선택(검색모드 토글) | `options=[{value,label}]`. 의미있는 옵션 기본 활성(하이브리드 RRF). BM25 전환 시 ScoreChip 라벨 `sim`→`bm25` |

### feedback (상태/신뢰)

| 컴포넌트 | 용도 | 핵심 props·예시 |
|---|---|---|
| **StatusPill** | 실행/문서/체크 상태(모노 라벨) | `tone=ok\|warn\|error\|running\|neutral`, `indicator=dot\|icon`. ok(teal)만 정상 상태에 색 |
| **ConnectionBadge** | `● localhost:1234 연결됨`(헤더 1급) | `status=ok\|...`, `endpoint`. **숨기지 말 것** |
| **ExternalCallsCounter** | `EXTERNAL CALLS: 0`(0=그린, 위반=레드) | `count` |
| **GateChip** | 게이트 통과/차단/폴백 | `state=pass\|fail\|fallback`. 예: `content empty → reasoning_content` |
| **ProgressBar** | 인덱싱/임베딩/평가 진행 | `value`, `tone`, `indeterminate`, `showLabel` |
| **Tooltip** | 호버/포커스 툴팁 | `label`, `side=top\|bottom\|left\|right` |

### data (데이터 표시) ★ docs-em 도메인 특화

| 컴포넌트 | 용도 | 핵심 props·예시 |
|---|---|---|
| **Metric** | 큰 모노 지표 + 라벨 + 부호 델타 | `label`, `value`, `delta`(자동 색), `invertDelta`(낮을수록 좋을 때: latency) |
| **ScoreChip** | 점수 KIND를 항상 라벨링(sim/bm25/rrf 혼동 방지) | `kind`, `value`, `gap`(음수=레드 보더), `decimals` |
| **ScoreBar** | red→green 평가 스케일 가로 미터 | `value`, `threshold`(합격선 틱), `max`, `showValue` |
| **DataTable** | 평가결과/청크/코퍼스 테이블 | `columns=[{key,label,mono,align,render}]`, `rows`, `onRowClick`, `dense` |
| **Tag** | ID/모델/코퍼스/메타 토큰(각진 모서리) | `icon`, `tone`, `onRemove` |
| **Kbd** | 단축키 캡 | children |

### layout / core

| 컴포넌트 | 용도 | 핵심 props |
|---|---|---|
| **Card** | 표면 컨테이너 | `header`, `footer`, `actions`, `inset`(코드/로그 웰), `padding` |
| **Tabs** | 언더라인 탭(teal 활성) | `items=[{value,label,count}]` |
| **Icon** | Lucide 라인 아이콘(1.75 stroke) | `name`, `size`, `color`. `Icon.names`에 목록. **이모지 금지** |

**자주 쓰는 아이콘**: search, send, file-text, database, layers, activity, gauge, check-circle, alert-triangle, x, rotate-cw, shield-check, trending-up/down, cpu, terminal, clock, filter, lock.

## 5. 3화면 UI 키트 (정본: `ui_kits/docs-em/`)

`index.html`로 인터랙티브 실행(헤더에서 화면 전환). 각 화면을 docs-em 표면 구현의 출발점으로 복사하세요.

| 파일 | 화면 | 핵심 구성 |
|---|---|---|
| `AppShell.jsx` | 공통 고정 헤더 + 사이드레일 | 로고·화면 네비 + 신뢰 클러스터(ConnectionBadge + `EXTERNAL CALLS: 0` + `100% LOCAL` — 3화면 고정, 숨김 금지) |
| `QueryScreen.jsx` | **A 질의응답** | 62/38 분할. 인라인 각주 `[¹][²]`(호버↔Top-k 하이라이트, 인용⊆Top-k), `재시도 1` 메타 아코디언, 검색모드 Segmented, Top-k=5 청크 카드(sim ScoreChip), 격차 -0.018, `cutoff —` 앰버 점선 |
| `IndexScreen.jsx` | **B 인덱싱** | 좌측 레일(nomic 768·norm1 활성, BGE-M3 1024 ⚠→DECISION 모달, LanceDB?/ChromaDB? 플레이스홀더), 드롭존+거부목록, 5노드 파이프라인, `prompt_tokens=0 (unreliable)`, 색인 테이블(손상스캔 파싱실패) |
| `EvalScreen.jsx` | **C 평가 대시보드** | 좌측 네비 + 실험 드롭다운(baseline/+bm25/+rrf), 거대 Recall@1 KPI(`[□■■■]`), 유사도 격차 KPI, before▸after 테이블, 통제 실험(단일 변수, 위반 ⚠), 안정성 게이지, 회귀 드릴다운, L1→L2→L3 타임라인. **+rrf 선택 시 Recall@1 `0 ▸ 1.0` 성공 데모**(red→green 플래시) |

> 키트는 **외형 재현**입니다 — 인터랙션은 가짜(화면 전환·모드 토글·실험 전환·호버·모달). 실제 모델/인덱스는 미연결.

## 6. 개발 시 적용 (실행 프롬프트 연계)

[07_실행프롬프트](../07_실행프롬프트/00_실행개요.md)로 docs-em 백엔드를 구현한 뒤, **UI를 붙이는 단계**에서 이 디자인 시스템을 씁니다.

1. **CSS 토큰 vendoring**: `디자인/docs-em Design System/tokens/`와 `styles.css`를 프론트엔드 프로젝트로 복사하고 `styles.css` 한 파일을 link.
2. **폰트 오프라인화 (폐쇄망 필수)**: `tokens/fonts.css`의 `src`가 CDN(jsdelivr·Google Fonts)을 가리킴 → **폐쇄망에서는 `.woff2`를 로컬(`assets/fonts/`)로 받아 src 재지정 필수.** (Pretendard, JetBrains Mono)
3. **컴포넌트 사용**: 각 `<Name>.prompt.md`로 props 확인 후 사용. 데이터는 전부 `.mono`(JetBrains Mono tabular-nums), UI 문구는 Pretendard.
4. **화면 출발점**: `ui_kits/docs-em/`의 3화면을 복사해 실제 `answer()`/인덱싱/`evaluate()` 결과에 배선.
5. **로고 교체**: `assets/logo-mark.svg`는 플레이스홀더 → 공식 마크로 교체.

### 불변식과의 정합 (UI에서도 지킬 것)

- **외부 API 0건(INV-3)**: 폰트·아이콘·번들 어디에도 런타임 외부 호출 없게. CDN 폰트는 빌드 타임에 vendoring.
- **신뢰 시그널 1급(고정)**: `localhost:1234`·`EXTERNAL CALLS: 0`·`100% LOCAL`은 3화면 헤더에 항상 노출.
- **정직한 수치**: Recall@1=0·격차 -0.018·`REGRESS`를 미화 없이. `prompt_tokens=0`엔 `(unreliable)` 앰버 태그(INV-4).
- **BGE-M3(1024)는 미채택 DECISION**: 화면 A에 노출 금지, 화면 B 레일에서만 ⚠+모달. (결정 상세: [00_결정 통합](../00_결정/00_결정해야할것_통합.md))
- **데모 식별자 주의**: `hr-0412`/`tr-0203`·`p.N`은 데모 전용 가짜값. 실제 chunk_id 스키마는 [03_DATABASE](../03_DATABASE/02_데이터모델_청크.md) 정본. 확정 수치(768·-0.018·0.687→0.695·12,972자·ctx 8192/65536·localhost:1234)만 사실.
- **데모 문서**: 휴가규정.pdf·출장경비.docx **2종만**.
