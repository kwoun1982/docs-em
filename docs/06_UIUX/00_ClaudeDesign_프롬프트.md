## ① 프로젝트 정의 · 톤

**docs-em**은 100% 로컬 전용 한국어 사내문서 RAG(검색증강생성) 시스템의 운영 도구다. 모든 추론·임베딩·생성은 로컬 LMStudio(`http://localhost:1234/v1`, OpenAI 호환)에서만 수행되며 **외부 클라우드 API 호출은 0건**이다. 이 시안은 3개 화면(A 질의응답, B 인덱싱, C 평가 대시보드)으로 구성된 **반응형 다크 웹 UI**다.

톤은 **폐쇄망 터미널 같은 차분한 다크 콘솔**이다. 장식을 걷어내고 실측 점수·지표가 스스로 말하게 한다. 낮은 채도 배경에 청록/그린(정상) 신호 하나만 또렷이 살리고, 회귀·재시도·실패 같은 위험 상태만 앰버/레드로 끌어올려 정보 밀도를 정직하게 유지한다. 데이터·식별자·점수는 모노스페이스, UI 문구는 산세리프. 과한 그라데이션·일러스트·바운스 모션은 금지한다. **레퍼런스: LM Studio 자체 UI, Linear, Vercel 대시보드, Grafana.**

## ② 글로벌 디자인 시스템

### 컬러 토큰 (정확히 이 값을 사용)

- **배경 Base**: `#0D1117` 최하단 캔버스 / `#0A0C10` 사이드바·코드블록(한 단계 더 가라앉힌 면)
- **표면 Surface**: `#161B22` 카드·패널 1단 / `#1C2128` hover·중첩 패널 2단
- **보더**: `#30363D` 기본 구분선 / `#3D444D` 강조 보더 / `#21262D` 미세 내부 분할선
- **텍스트**: `#E6EDF3` 본문 1차 / `#9DA7B3` 보조·라벨 / `#6E7681` 비활성·플레이스홀더
- **정상(청록/그린)**: `#2DD4BF` 청록(연결 정상·강조 액션) / `#3FB950` 그린(성공·Recall 달성·점수 상승). 정상 틴트 `#2DD4BF1A`
- **경고(앰버)**: `#D29922` (재시도·thinking 잘림·컷오프 근접·토큰 신뢰불가). 틴트 `#D299221A`
- **실패·회귀(레드)**: `#F85149` (Recall@1=0 회귀·연결 끊김·근거 부족·실패). 틴트 `#F851491A`
- **정보(블루)**: `#58A6FF` (중립 링크·메타정보) — 상태 3색과 분리하여 혼동 방지

### 타이포그래피

- **UI 산세리프 (Pretendard / Inter)**: 메뉴·버튼·설명·화면 제목·한국어 본문. line-height 1.6, 자간 0
- **모노스페이스 (JetBrains Mono / IBM Plex Mono)**: 유사도 점수(`0.687`), 격차(`-0.018`), 차원(`768`/`1024`), 컨텍스트 길이(`8192`/`65536`), chunk_id, 문서 위치, `localhost:1234`, 모델명, `finish_reason` 등 **모든 데이터·식별자·코드**
- 숫자는 모노 `font-variant-numeric: tabular-nums` 고정(점수 세로 정렬·카운트업 자릿수 흔들림 방지)
- **위계**: 화면 제목 20–24px / 섹션 16px / 본문 14px / 데이터·라벨 12–13px 모노. **1급 지표(Recall@1 등)는 28–32px 모노로 단독 강조**
- 장문 한국어를 모노로 표기하지 말 것(가독성 저하 방지)

### 공통 컴포넌트

- **상태 뱃지**: 12px 모노·라운드 6px·좌측 점(dot)+라벨. 정상=그린/청록 점, 경고=앰버 점, 실패=레드 점. **색 외에 점·아이콘·텍스트 라벨로 이중 구분**
- **점수 칩(Score chip)**: 모노 tabular-nums, `sim 0.695` 형태로 라벨+값 병기. 격차 음수면 레드 테두리, 양수면 그린 테두리. **벡터/하이브리드 모드에서만 코사인 `sim` 접두; BM25 단독 모드에서는 `bm25 12.4`처럼 점수 종류 라벨을 바꿔 코사인과 혼동 금지**
- **청크 카드**: Surface 1단 배경 + 좌측 4px 상태 바(인용됨=청록, 컷오프 미만=흐림 opacity 0.45). 헤더에 문서명(산세리프)+chunk_id(모노)+점수 칩, 본문 미리보기 2~3줄, 인용 시 청록 보더 강조
- **컷오프 임계선**: Top-k 패널·차트 공통. score 컷오프는 **수평 점선(앰버, 1px dashed) + 우측 라벨 `cutoff`**로 렌더. 임계선 아래(컷오프 미만) 청크는 흐림 처리. **임계값 자체는 DECISION 미확정이므로 데모에서는 #5 아래 가상의 점선으로만 표기하고 구체 숫자 라벨은 비움(`cutoff —`)**
- **연결상태 인디케이터**: 헤더 고정. `● localhost:1234` 청록=정상 / 앰버=지연 / 레드=끊김. 모노 표기 + 폐쇄망 락 아이콘 동반. **지연(앰버) 트리거 = 폴링 응답이 느려 마지막 헬스체크가 stale인 상태(정상=응답, 끊김=무응답, 지연=응답 지연); 임계 초 단위는 DECISION 미확정이라 데모에서 정상 상태로 표기**
- **게이트 표시**: 통과=그린 체크 / 폴백 발동=앰버 펄스 / 차단=레드. `content empty → reasoning_content fallback`처럼 모노로 사유 표기

### 데이터 시각화

- **유사도 격차 막대**: 정답 vs 오답 수평 막대 쌍. 회귀 시 정답이 짧고 `-0.018` 격차를 레드 라벨링. 역전 성공 시 정답이 1위로 그린 채움
- **Recall 게이지**: Recall@1/3/5/10을 0–1 게이지로. `0`은 레드, `1` 달성 순간 그린. **Recall@1을 가장 크게 단독 배치**
- **before·after 비교**: 좌(현재)·우(실험 후) 2열, 변경 변수 1개를 상단 모노로 명시. 개선=그린 델타, 악화=레드 델타
- **차트 절제**: 그리드 최소, 축 라벨 모노 tabular-nums, 그라데이션 금지·단색 채움. 임계선(컷오프)은 앰버 점선

### 신뢰 시그널

- 헤더 1급 시민: `100% LOCAL` 청록 뱃지 + 폐쇄망 락 아이콘 상시 노출
- `EXTERNAL CALLS: 0` 모노 카운터 — 0이면 그린, 1 이상이면 즉시 레드 위반 경고
- `localhost:1234` 연결 패널: 엔드포인트·모델명·차원(`768`)·정규화(`norm=1.0`)를 모노 키-값 리스트로
- **정직성 주석**: 신뢰 불가 데이터에 앰버 인라인 태그 — `prompt_tokens=0 (unreliable)`. 수치 미화 금지

### 모션 · 접근성

- 파이프라인 세그먼트 채움 200–300ms ease-out, 진행 세그먼트만 청록 펄스(1.2s loop). 점수 카운트업 400ms tabular-nums 롤업(단조 증가만). 회귀 0→1 전환은 보더 레드→그린 1회 플래시. **패럴랙스·바운스·장식 모션 금지**, `prefers-reduced-motion` 시 최종 상태만 즉시 표기
- 본문 대비 WCAG AA(4.5:1) 이상. **색만으로 상태 구분 금지** — 색+형태(점·체크·X)+텍스트 라벨(`OK`/`RETRY`/`FAIL`) 병기. 포커스 링은 청록 2px 아웃라인
- **반응형 분기점**: 데스크톱 ≥1200px 기준. `<1024px`에서 화면 A 우측 Top-k 패널은 답변 아래로 스택, 화면 B·C 좌측 사이드바는 상단 햄버거 메뉴로 접힘. `<768px`에서 화면 C 통제실험/안정성 2열 비교 카드는 1열로 세로 적층

## ③ 화면별 상세 명세

### 화면 A — 질의응답 + 출처 인용 (일반 사용자, 핵심 화면)

**레이아웃**: 풀폭 고정 헤더 + 좌측 사이드바 없음(단일 검색 집중). 메인 영역 **62%**(채팅 스레드 + 하단 고정 입력창), 우측 패널 **38%**("근거 청크 Top-k", 답변과 같은 높이). 좁은 뷰포트(`<1024px`)에서는 우측 패널이 답변 아래로 스택.

```
┌─────────────────────────────────────────────────────────────────────────┐
│ HEADER  docs-em ▸ 사내문서 검색   [●LOCAL] localhost:1234 연결됨  외부:0건  │
│  GEN: gemma-4-e4b · ctx 65536 ▾   EMB: text-embedding-nomic… · 768d        │
├──────────────────────────────────────────┬──────────────────────────────┤
│ MAIN (62%)                                │ PANEL: 근거 청크 Top-k (38%)  │
│ [검색모드: ●하이브리드 RRF ○벡터 ○BM25]   │ Top-k 결과 (k=5)  RRF 정렬    │
│ ▣ 질의 "연차 휴가 신청 절차"               │ ┌─#1 출장경비.docx · p.2 ✕─┐ │
│ ▣ 답변 [재시도1ⓘ ▾접힘]                    │ │ tr-0203   sim 0.713 ✕오답 │ │
│   연차 휴가는 신청서 작성 후 팀장 승인을    │ ├─#2 출장경비.docx · p.5 ──┤ │
│   거쳐 신청합니다 [¹]. 신청은 사용 7일      │ │ tr-0511   sim 0.701  —    │ │
│   전까지 제출합니다 [²].                    │ ├─#3 휴가규정.pdf · p.4 ───┤ │
│   출처                                     │ │ hr-0412 ▲정답 sim 0.695   │ │
│   [¹] 휴가규정.pdf · p.4 · hr-0412         │ │  격차 -0.018 ▼ (vs #1)    │ │
```

**컴포넌트**
- 헤더 연결 인디케이터: 청록 점 + `LOCAL` 뱃지 + `localhost:1234 연결됨` + `외부 네트워크 0건`(폐쇄망 1급 시민). 끊김 시 레드 점 + `연결 끊김`. **헤더 폭 부족 시 숨김 우선순위: EMB 모델 풀네임 축약(`text-embedding-nomic…`) → GEN ctx 라벨 → 그래도 부족하면 GEN 모델명 말줄임. `LOCAL` 뱃지·`외부:0건`·`localhost:1234`는 폐쇄망 신뢰 시그널이므로 절대 숨기지 않음**
- 모델 표기: `GEN: gemma-4-e4b · ctx 65536`(드롭다운 — `qwen3.6-35b-a3b · ctx 8192` / `qwen3.5-9b · ctx 8192` 선택, 각 항목에 ctx 노출하여 기본 모델 gemma의 긴 컨텍스트 우위를 드러냄. 파라미터 수는 노출하지 않음), `EMB: text-embedding-nomic-embed-text-v1.5 · 768d`(읽기 전용 모노, 재정규화 금지 함의 — **모델명은 반드시 `text-embedding-` 접두 포함 풀네임, 헤더 폭 부족 시 말줄임 `text-embedding-nomic…`으로 축약하되 다른 모델명처럼 보이게 잘라내지 말 것**)
- 검색 모드 토글: 세그먼티드 컨트롤 `벡터 / BM25 / 하이브리드 RRF`, **기본 하이브리드 RRF 활성**(청록 밑줄). BM25 단독 모드 전환 시 점수 칩 라벨이 `sim`(코사인)에서 `bm25`(BM25 점수)로 바뀜
- 질의 입력창: 하단 고정 멀티라인 + `전송`. 위에 예시 칩 `연차 휴가 신청 절차`(회귀 케이스), `출장경비 정산` — 클릭 시 입력창 자동 채움(즉시 전송 아님)
- 답변 본문: 산세리프, 문장 끝 인라인 각주 위첨자 `[¹][²]` — 호버 시 우측 청크 하이라이트. 하단 "출처" 블록에 `문서명 · 위치 · chunk_id`(모노). **각주가 인용하는 chunk_id는 반드시 우측 Top-k 패널에 존재하는 청크여야 함(인용 ⊆ 전달된 청크 집합 불변식). 데모에서는 `hr-0412`(#3)·`hr-0712`(#5)만 인용**
- 메타 상태 배지: `재시도 1 ⓘ`(기본 접힘, 낮은 채도) — 펼치면 `content 빈값 → reasoning_content 폴백`, `<think> 정제됨`, `finish_reason=length 재시도 1회`
- Top-k 근거 패널: 카드 리스트 — `#순위`, `문서명 · 위치`, `chunk_id`(모노), `sim 0.###`(소수 셋째자리), 스니펫 2줄. 정답=`▲정답`(그린), 오답=`✕오답`(레드). **하단에 컷오프 점선 + `cutoff —` 라벨, 점선 아래 청크는 흐림(데모에서는 컷오프 미만 청크 없음, 점선만 표기)**
- 회귀 강조 바: 정답(#3 휴가규정 hr-0412) vs 1위 오답(#1 출장경비 tr-0203) 사이 `격차 -0.018` 앰버 + 작은 수평 점수 막대. **정답 문서(휴가규정)의 첫 히트가 3위 = MRR 0.333과 정합**

**상태·엣지케이스**
- 로딩: `검색 중 → 생성 중` 단계 텍스트 + 청록 펄스. 우측 패널 청크 카드 스켈레톤
- 빈 상태(첫 진입): 중앙에 예시 칩 2개 + `한국어로 질문하세요`, 우측 패널 `근거 청크가 여기에 표시됩니다`

### 화면 C — 검색 품질 평가 대시보드 (평가자)

**컴포넌트**
- **KPI 카드 ① 회귀 Recall@1**: 거대 모노 숫자 `0`, `▸ 목표 1`, 게이트 칩 `L1 G1 미달` + 레드 `REGRESS`. **하단 4칸 진행 인디케이터 = Recall@1/@3/@5/@10 각 칸(달성=채움 그린, 미달=빈칸). 현재 데모: Recall@1 미달(빈칸)·@3/@5 달성(채움)·@10 달성(채움) → `[□■■■]`**. **화면에서 가장 큰 요소**
- **KPI 카드 ② 유사도 격차**: 모노 `-0.018`(레드), `▸ 목표 +양수 역전`. 음수→양수 전환 시 청록 `PASS` 게이트 라벨
- 골든셋 전체 지표 패널: Recall@1/@3/@5/@10·MRR·nDCG@3 6행, 각 행 before/after 막대 2개 + 델타 칩(▲청록/▼레드), 모노 점수
- 통제 실험 비교 카드: 실험 3개(임베딩 교체 / BM25 추가 / RRF)별 변경 변수 수 칩 + 단일 변수 기여도. **변수 1개=청록 ✔, 2개 이상=앰버 ⚠ "통제 위반"**
- 회귀 드릴다운 테이블: 질의 / 정답문서 / 정답순위 / 정답점수 / 최상위 오답문서 / 격차 / 게이트. "연차 휴가 신청 절차" 행은 정답=휴가규정·**문서 첫 히트 순위 `3`**·오답=출장경비·격차 `-0.018` 하이라이트
- 실험 이력 타임라인: L1→L2→L3 노드에 핵심 지표(Recall@1, MRR) 추이 라인 + 게이트 통과 마커
- 안정성 보조지표 카드: `content 빈값 발생률`, `length 재시도율` 미니 게이지(낮을수록 좋음, 청록=낮음/앰버=높음). **데모값 = `content 빈값 8%`(청록), `length 재시도 12%`(청록). 임계 ≥25%부터 앰버**

**상태·엣지케이스**
- 로딩: KPI/테이블 모노 스켈레톤 + "골든셋 N건 평가 중…"
- 빈 상태: 미실행 시 KPI `—` + "베이스라인 평가를 먼저 실행하세요"
- 회귀 미해결(현재): Recall@1=0, 격차 음수 → 레드 `REGRESS` + 게이트 G1 미달 유지
- 회귀 해결: 정답 1위 + 격차 양수 → 청록 `PASS`, G1 통과 마커 점등
- 통제 위반: 변수 2개 이상 동시 변경 감지 시 앰버 ⚠ "단일 변수 원칙 위반 — 기여도 해석 불가"
- 안정성 경고: length 재시도율/content 빈값률 임계(≥25%) 초과 시 앰버 게이지 + "reasoning_content 폴백 점검"
- 연결 끊김: 헤더 점 레드 + "로컬 서버 미응답" 토스트(외부 호출 시도 절대 없음)

**데이터 필드**

## ⑤ 산출물 요구

1. **반응형 다크 웹 UI**. 데스크톱(≥1200px) 기준 설계, `<1024px`에서 화면 A 우측 패널은 답변 아래로 스택·화면 B·C 사이드바는 햄버거로 접힘, `<768px`에서 화면 C 2열 카드는 1열로 적층.
2. **3화면 네비게이션**: 검색(A) / 인덱싱(B) / 평가(C) 전환. 헤더의 연결 상태·폐쇄망 뱃지는 3화면 공통 고정.
3. **정직한 데이터 표현**: 위 데모 수치를 그대로 표기. Recall@1=0, 격차 -0.018 같은 부정적 실측을 미화 없이 노출하고, 신뢰 불가 값(`prompt_tokens=0`)에는 앰버 주석.
4. 모든 데이터·식별자·점수는 모노 tabular-nums, UI 문구는 산세리프. 컬러 토큰·타이포 위계·모션/접근성 규칙을 정확히 적용.
5. 회귀 케이스 0→1 전환 강조 패널을 화면 C에 1급으로 배치(레드→그린 1회 플래시 연출). **위 ④의 "개선 성공 데모 세트"(Recall@1 0▸1, 격차 -0.018▸+0.012, MRR 0.333▸1.0, nDCG@3 0.5▸1.0, 순위 3▸1)를 after=성공 상태 시안으로 사용.**

## ⑥ 하지 말 것

- **외부 클라우드 API 흔적 금지**: OpenAI / Anthropic / Cohere / Gemini 등 어떤 외부 서비스 로고·URL·키·모델명도 노출 금지. 연결 대상은 오직 `localhost:1234`. `EXTERNAL CALLS: 0` 유지.
- **로그인·회원가입·권한·결제 화면 금지**. 이 도구는 폐쇄망 내부 운영 도구다.
- **과한 그라데이션·일러스트·바운스/패럴랙스 모션 금지**. 단색 채움·절제된 트랜지션만.
- **없는 기능 발명 금지**: 명세에 없는 차트·지표·버튼·설정을 추가하지 말 것. 특히 BGE-M3 1024차원은 **미채택 DECISION**이므로 화면 A에는 노출하지 말고, 화면 B 사이드바에서만 `⚠`+DECISION 모달로 표기.
- **수치 날조 금지**: 768/1024차원, -0.018, Recall@1 0→1, 12,972자, 0.687→0.695, ctx 8192/65536, localhost:1234 외의 새 수치를 지어내지 말 것. (안정성 게이지 `8%`/`12%`·임계 `25%`·개선 데모 `+0.012`는 데모 채움용 표기로 한정하며, 실측이 아닌 데모값임을 카드에 명시.)
- **재정규화 표기 금지**: 임베딩은 이미 L2 정규화(norm=1.0)되어 있으므로 "재정규화" 액션 버튼을 만들지 말 것(읽기 전용 표기만).
- **데모 문서·식별자 발명 주의**: 회귀 케이스 문서는 `휴가규정.pdf`·`출장경비.docx` **2종만** 사용(`복무규정.pdf` 등 제3 문서 추가 금지). chunk_id는 데모 전용 가짜값임을 ④ 주석대로 취급.
- **인용 부분집합 불변식 준수**: 답변 각주가 인용하는 모든 chunk_id는 우측 Top-k 패널에 실제로 존재하는 청크여야 함(인용 ⊆ 전달된 청크 집합). Top-k에 없는 출처를 각주로 표기 금지.
