## 사용 방법

1. [00_ClaudeDesign_프롬프트.md](./00_ClaudeDesign_프롬프트.md)를 연다.
2. 최상단 인용 블록(`>` 안내) 아래의 **`## ① 프로젝트 정의 · 톤`부터 문서 끝까지** 전체를 복사한다.
3. Claude Design 입력창에 붙여넣고 실행한다.
4. 생성된 시안에서 화면 전환(A 검색 / B 인덱싱 / C 평가)과 회귀 케이스 0→1 전환 패널을 확인한다.

## 무엇을 설계했나

기획서 3개 액터(일반 사용자/운영자/평가자)에 대응하는 3화면 통합 콘솔:

- **화면 A — 질의응답 + 출처 인용** (핵심): 한국어 질의 → 답변 → 인라인 각주 출처 + 우측 Top-k 근거 청크 패널. 검색 모드 토글(벡터/BM25/하이브리드 RRF), reasoning 폴백·`<think>` 정제·length 재시도 메타 상태, 근거 부족 처리.
- **화면 B — 인덱싱 콘솔**: PDF/DOCX 드롭존, 5단계 파이프라인(파싱→문장분리→청킹→임베딩→저장) 진행, 색인 문서 테이블, 멱등 재색인, nomic↔BGE-M3 모델 토글(DECISION 모달).
- **화면 C — 평가 대시보드**: 회귀 케이스 Recall@1(0→1) KPI, 골든셋 지표(Recall@k/MRR/nDCG) before/after, 통제 실험(변수 1개씩), 회귀 드릴다운, 생성 안정성 보조지표.

## 톤

폐쇄망 개발 도구 — 다크 콘솔 기반, 데이터/점수는 모노스페이스, "100% 로컬 / 외부 호출 0건" 신뢰 시그널을 1급으로 노출. 레퍼런스: LM Studio UI · Linear · Vercel 대시보드 · Grafana.

## 생성 방식 (참고)

이 프롬프트는 멀티 에이전트 워크플로우로 생성·검증됐습니다.

1. 3화면 UI 명세를 병렬 설계 + 공통 디자인 시스템 도출
2. 단일 Claude Design 프롬프트로 종합
3. 3개 독립 렌즈(기획 충실성 / 실측 불변식·환각 / 디자인 완결성)로 어드버서리얼 검증 → 34개 지적을 1회 반영

검증에서 잡아낸 핵심 수정: 인용 부분집합 불변식 위반(존재하지 않는 청크 ID 인용) 제거, ASCII 목업과 데모 표의 Top-k 구성 일치, 정답 문서 3위·MRR 0.333 정합(INV-7), 발명된 제3 문서명 제거, 컷오프 임계선·안정성 게이지 데모값 보강.

## 주의

- 모든 확정 수치(`768`/`1024`차원, `-0.018`, Recall@1 `0→1`, `0.687→0.695`, ctx `8192`/`65536`, `localhost:1234`)는 [시작프롬프트.md](../../시작프롬프트.md) SSOT와 [99_AI참조/01_실측제약_불변식.md](../99_AI참조/01_실측제약_불변식.md) INV-1~8을 정본으로 합니다.
- 프롬프트 내 `chunk_id`(`hr-####`/`tr-####`)·페이지 위치·안정성 게이지 백분율은 **화면을 채우기 위한 데모 전용 가짜값**입니다. 실제 청크 ID 스키마는 [03_DATABASE](../03_DATABASE/00_벡터스토어_스키마.md) 정본을 따릅니다.
