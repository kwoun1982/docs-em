## 1. 문제 정의 (Problem Statement)

사내 한국어 문서(규정·정책·매뉴얼 등)를 자연어로 질의했을 때, **현재 임베딩 모델이 정답 문서를 1순위로 검색하지 못하는 품질 결함**이 핵심 문제입니다.

### 1.1 실측된 회귀 케이스

가장 명확한 실패 사례는 다음과 같습니다.

| 항목 | 값 |
|---|---|
| 질의 | "연차 휴가 신청 절차" |
| 정답 문서 | 휴가규정 |
| 정답 순위 | **3위** |
| 오답 문서 | 출장경비 (정답보다 상위) |
| 정답−오답 유사도 격차 | **−0.018** (정답이 더 낮음) |
| Recall@1 | **0** |

즉, 의미상 명백히 정답인 문서가 무관한 문서보다 낮은 유사도를 받아 1순위 검색에 실패합니다. 이는 단일 사례가 아니라 **한국어 임베딩 품질의 구조적 약점**을 드러내는 대표 회귀 케이스입니다. (근거: [시작프롬프트.md](../../시작프롬프트.md) §B-12, [불변식.md](../99_AI참조/01_실측제약_불변식.md) INV-7)

### 1.2 제약 조건에서 비롯된 문제

- 외부 클라우드 임베딩/생성 API(OpenAI/Anthropic/Cohere)를 **사용할 수 없는** 로컬 전용 환경입니다. → [시작프롬프트.md](../../시작프롬프트.md) §A-1·§A-2 ①·§B-12, [불변식.md](../99_AI참조/01_실측제약_불변식.md) INV-3
- 따라서 상용 고품질 임베딩으로의 우회가 불가능하며, **로컬에서 구동 가능한 모델만으로 한국어 검색 품질을 끌어올려야** 합니다.

## 2. 목표와 비목표 (Goals / Non-Goals)

### 2.1 목표 (Goals)

| 우선순위 | 목표 |
|---|---|
| **1순위** | **한국어 문서 검색 품질 향상** (회귀 케이스 Recall@1 0→1 달성) |
| 2순위 | 검색된 근거에 기반한 한국어 답변 생성 + 출처 인용 |
| 3순위 | 품질을 정량 측정·회귀 방지하는 평가 하네스 구축 |
| 4순위 | 100% 로컬 구동 (외부 네트워크 의존 0) |

> ✅ **확정 (2026-06-17)** — 주 사용 언어는 **한국어**입니다(§B-9 #1, 사용자 결정). 이 확정 위에서 1순위 목표(한국어 검색 품질)가 성립하며, BGE-M3 임베딩 교체 검토 경로(§B-5)가 활성화됩니다(채택 자체는 측정 후 결정 — §9 참조).

### 2.2 비목표 (Non-Goals)

- 외부 클라우드 LLM/임베딩 API 연동 (영구 금지 — §A-2 ①·§B-12, INV-3)
- 멀티모달(이미지·표 OCR·음성) 검색 — 본 단계 범위 밖 [확인필요]
- 멀티유저 권한관리·SaaS화·웹 서비스 배포 — 범위 밖
- 모델 자체 파인튜닝/사전학습 — 교체·하이브리드·리랭킹으로 해결
- 비영어/비한국어 다국어 최적화 — 한국어 1순위에 집중
- e5 계열 등 **다른 임베딩 모델로 교체 시** prefix 규칙이 nomic과 달라집니다. 예: e5 계열은 `query:`/`passage:` prefix가 **필수**이며 누락 시 품질이 급락합니다. 본 PRD의 prefix 미미 판정(§4.1)은 현재 nomic 모델에 한정된 사실이므로, 모델 교체 시 prefix 규칙 재검증을 전제로 합니다. (근거: §B-5, INV-8)

### 4.3 답변 생성 (Generation)

| ID | 요구사항 | 우선순위 | 추적(INV) |
|---|---|---|---|
| FR-GEN-1 | 검색 청크를 컨텍스트로 한국어 답변 생성 | P0 | — |
| FR-GEN-2 | `content` 빈값 시 `reasoning_content` 폴백 | P0 | INV-2 |
| FR-GEN-3 | `<think>…</think>` 블록 정제 후 최종 답변 추출 | P0 | INV-5 |
| FR-GEN-4 | `finish_reason=length` 잘림 시 재시도 | P0 | INV-6 |
| FR-GEN-5 | **무효 우회책 4종을 구현하지 않는다**(부정 요구): `/no_think`·`enable_thinking=false`·`reasoning_effort=low`·`max_tokens=4000`은 실측 무효이므로 코드에 넣지 않음 | P0 | INV-2 |

> **BLOCKING 이슈.** 로컬 생성모델 3종(qwen3.6-35b-a3b / qwen3.5-9b / gemma-4-e4b) 모두 `message.content`가 빈 문자열로 오고 실제 답은 `message.reasoning_content`로 옵니다. `/no_think`·`enable_thinking=false`·`reasoning_effort=low`·`max_tokens=4000`이 **전부 무효**이며, 무한 사고로 `finish_reason=length` 잘림(한 문장 질문에 12,972자)이 발생합니다. 따라서 **content 빈값 폴백 + `<think>` 정제 + length 재시도**가 답변 생성의 필수 전제입니다. → §A-2 ③·§B-4·§B-7, INV-2/5/6

**FR-GEN-3 수용 기준(정제 규칙).** `reasoning_content`(또는 content)에서 `<think> … </think>` 패턴을 제거하고 그 뒤(또는 외부)의 최종 답변 텍스트만 추출합니다. 합격 판정: 최종 출력에 `<think>`/`</think>` 토큰 및 사고과정 본문이 포함되지 않아야 합니다. 정제·추출 함수 계약은 [04_아키텍처_API/02_내부API_인터페이스.md](../04_아키텍처_API/02_내부API_인터페이스.md)를 정본으로 참조합니다.

**FR-GEN-4 수용 기준(재시도 정책).** `finish_reason=length`인 경우 **더 높은 `max_tokens`로 1회 재시도**하거나 **stop 토큰을 주입**합니다(SSOT §B-4 ③). 합격 판정: 재시도 후에도 length로 잘리면 실패로 기록하여 재시도 실패율 지표(§6.2)에 반영합니다.

생성모델 사양(모두 로컬, `parallel=4`):

| 모델 | 파라미터/구조 | 컨텍스트 |
|---|---|---|
| qwen/qwen3.6-35b-a3b | 35B MoE | 8192 |
| qwen/qwen3.5-9b | 9B [추정] | 8192 |
| google/gemma-4-e4b | 4B [추정] | 65536 |

> 출처 구분: **컨텍스트 값(8192 / 8192 / 65536)과 35B MoE는 실측값**입니다. 파라미터 수 **9B / 4B만** 모델명에서 유추한 `[추정]`입니다(컨텍스트에는 `[추정]`이 붙지 않습니다). (근거: §B-3, §B-12)

> **DECISION** — 답변 생성 기본 모델은 **gemma-4-e4b(ctx 65536)** 를 권장합니다(§B-9 #8). 긴 컨텍스트 RAG에 유리하나 4B급이라 답변 품질 한계가 있을 수 있어, BLOCKING(§B-4) 해결 후 재평가가 필요합니다. settings에서 모델 스위칭이 가능하도록 둡니다. 최종 선택은 사용자 결정 사항입니다.

## 7. 해결 방향 요약 (참조용)

본 PRD는 요구사항을 정의하며, **구현 방법은 각 전담 문서로 링크**합니다.

| 문제 | 해결 후보 | 결정/상태 |
|---|---|---|
| 한국어 검색 품질 | 임베딩 교체(BGE-M3: 1024차원, ctx8192, Dense+Sparse+ColBERT, MIT, 한국어 Recall@1 우수) | GGUF 1~2GB 다운로드·적용은 **DECISION**(§B-9 #2) |
| 단일 모델 한계 | 하이브리드(벡터+BM25, RRF) + 리랭킹 | 스택 미확정 |
| 생성모델 BLOCKING | reasoning 폴백 게이트(폴백+정제+재시도) | 필수 구현 (INV-2/5/6) |

> **BGE-M3 교체 시 핵심 파급효과(리스크).** 차원이 **768→1024로 변경되어 인덱스 전체 재생성**이 필요하고, 새 모델의 출력 norm을 **재측정하여 INV-1(재정규화 금지) 가드를 갱신**해야 합니다. 또한 03_DATABASE 스키마의 차원 정의와 05_운영_런북 §2 재인덱싱 절차를 함께 갱신해야 합니다. (근거: §B-5, [불변식.md](../99_AI참조/01_실측제약_불변식.md) INV-1, [03_DATABASE/00_벡터스토어_스키마.md](../03_DATABASE/00_벡터스토어_스키마.md), [05_운영_평가/01_운영_런북.md](../05_운영_평가/01_운영_런북.md))
