## 1. Python 가상환경 (uv)

uv로 가상환경을 만들고 Python 3.11을 고정합니다.

```bash
# uv 미설치 시 (macOS/Linux)
curl -LsSf https://astral.sh/uv/install.sh | sh

# 프로젝트 루트에서
cd /Users/kykwoun/__DEV/docs-em

# Python 3.11 가상환경 생성
uv venv --python 3.11

# 활성화 (macOS/Linux, zsh/bash)
source .venv/bin/activate

# 버전 확인 (3.11.x 여야 함)
python --version
```

> **Windows [확인필요]**: uv 설치는 PowerShell에서 `powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"`, 가상환경 활성화는 `.venv\Scripts\activate`입니다. 본 문서의 스모크 테스트(§4)는 macOS/Linux 셸 기준이라 Windows에서는 명령 변환이 필요합니다(0-1표 'Windows [확인필요]'와 연결).

> **pip 대안**: uv를 쓰지 않는다면 `python3.11 -m venv .venv && source .venv/bin/activate`(Windows는 `.venv\Scripts\activate`)로 대체할 수 있습니다. 이후 설치 명령은 `uv pip install ...` → `pip install ...`로 바꿔 읽으세요.

> **`python` vs `python3` 주의**: 일부 환경(예: macOS)은 PATH에 `python`이 없고 `python3`만 존재합니다. **위 venv를 활성화한 세션 안에서는 `python`이 3.11로 해석**되므로, 본 문서의 모든 스모크 테스트는 **venv 활성 상태를 전제**합니다. venv 비활성 상태로 테스트 블록만 단독 복붙하면 `command not found: python`이 날 수 있으니, 그 경우 `python` → `python3`로 바꿔 실행하세요.

## 2. 의존성 설치

> 스택은 **미확정**입니다(시작프롬프트.md §B-6·§B-9). 아래는 권장 기본값이며, **DECISION** 항목(벡터스토어·BM25·임베딩 교체)은 01_기술스택결정서.md *(작성 예정)* 에서 확정한 뒤 설치하세요. 미확정 항목까지 한꺼번에 깔 필요는 없습니다.

| 계층 | 패키지 | 역할 | 단계 | 비고 |
| --- | --- | --- | --- | --- |

| 문서 파싱 | `pymupdf`, `python-docx` | PDF / DOCX 로딩 | L1 | `markitdown`은 대안 |

- **리랭커**(`BGE-reranker-v2-m3`)는 L3 항목이며 **로컬 서빙 방식이 미확인**입니다. LMStudio가 cross-encoder를 OpenAI 호환 API로 서빙한다는 보장이 없어, 필요 시 `FlagEmbedding`/`sentence-transformers` 별도 프로세스가 추가될 수 있습니다. → 시작프롬프트.md §B-2 ⑥ 참조. **이 단계에서는 설치하지 않습니다.** [확인필요]

명령형 설치 대신 선언형으로 관리하려면 루트에 `pyproject.toml`을 둡니다.

```toml
[project]
name = "docs-em"
version = "0.1.0"
description = "로컬 한국어 문서 RAG 시스템"
requires-python = ">=3.11"

    "openai",
    "python-dotenv",

[project.optional-dependencies]
hybrid = ["kiwipiepy", "rank_bm25"]   # L2 하이브리드 검색
```

```bash
# 선언형 설치

# L2 진입 시
uv pip install -e ".[hybrid,dev]"
```

## 3. `.env` 설정

비밀값은 코드에 하드코딩하지 않습니다. 단, **로컬 LMStudio는 API 키가 불필요**하므로 더미 값을 씁니다(시작프롬프트.md §A-1).

### 3-1. `.env.example` 생성

루트에 `.env.example`을 만들고, 복사해서 `.env`로 씁니다(`.env`는 `.gitignore`).

```bash
# .env.example
LMSTUDIO_BASE_URL=http://localhost:1234/v1
LMSTUDIO_API_KEY=lm-studio          # 더미 값 (로컬은 키 불필요)

# 모델 식별자 (LMStudio에 로드된 이름과 정확히 일치해야 함)
EMBEDDING_MODEL=text-embedding-nomic-embed-text-v1.5
GEN_MODEL=google/gemma-4-e4b        # 기본 생성모델 (DECISION §B-9 #8)
```

```bash
cp .env.example .env
```

### 3-2. 환경변수 표

| 변수 | 기본값 | 설명 |
| --- | --- | --- |
| `LMSTUDIO_BASE_URL` | `http://localhost:1234/v1` | OpenAI 호환 엔드포인트. 실측 고정값 |
| `LMSTUDIO_API_KEY` | `lm-studio` | 더미. 로컬은 인증 없음 |
| `EMBEDDING_MODEL` | `text-embedding-nomic-embed-text-v1.5` | 768d·L2정규화됨. 교체 시 인덱스 재생성(§4-3) |
| `GEN_MODEL` | `google/gemma-4-e4b` | ctx 65536. **DECISION**(§B-9 #8) — qwen 계열로 스위칭 가능 |

> 모델명은 LMStudio에 실제 로드된 이름과 **정확히** 일치해야 합니다. `GET /v1/models`로 확인하세요(§4-2). 인덱싱 시 기록되는 `model`·`dim`·`normalized`·`metric` 메타 필드의 정식 스키마는 [벡터스토어 스키마](../03_DATABASE/00_벡터스토어_스키마.md)에 정의돼 있습니다.

## 4. LMStudio 모델 로드 및 점검

### 4-1. 서버 켜기와 모델 로드

1. LMStudio 실행 → **Developer / Local Server** 탭에서 서버를 **Start**합니다(포트 `1234` 기본).
2. 필요한 모델을 **Load**합니다:
   - 임베딩: `text-embedding-nomic-embed-text-v1.5` (768d, 출력 이미 L2 정규화됨)
   - 생성: `qwen/qwen3.6-35b-a3b`(35B MoE, ctx 8192) / `qwen/qwen3.5-9b`(ctx 8192, 9B [추정]) / `google/gemma-4-e4b`(ctx 65536, 4B [추정]) 중 사용할 것. 모두 `parallel=4` 전제.
3. **임베딩과 생성 모델을 동시에** 로드해 둬야 인덱싱·검색·생성을 한 세션에서 돌릴 수 있습니다.

> 모델별 ctx·파라미터 수(9B/4B는 [추정])·실측 제약 상세는 [02_로컬환경_LMStudio실측.md](./02_로컬환경_LMStudio실측.md)(시작프롬프트.md §B-3·§B-12)에서 다룹니다. 여기서는 셋업 절차만 봅니다.

### 4-2. 연결·임베딩 스모크 테스트

서버가 살아있는지, 임베딩 스펙이 실측과 맞는지 즉시 확인합니다. **(아래 블록은 §1 venv 활성 상태를 전제로 합니다. venv 미활성 시 `python` → `python3`로 바꿔 실행하세요.)**

```bash
# 로드된 모델 목록
curl -s http://localhost:1234/v1/models

# 임베딩 1건 호출 — 차원·norm 직접 확인
curl -s http://localhost:1234/v1/embeddings \
  -H "Content-Type: application/json" \
  -d '{"model":"text-embedding-nomic-embed-text-v1.5","input":"연차 휴가 신청 절차"}' \
  | python3 -c "import sys,json,math; v=json.load(sys.stdin)['data'][0]['embedding']; print('dim=',len(v),'norm=',round(math.sqrt(sum(x*x for x in v)),4))"
```

**기대 출력(이진 판정)**: `dim= 768 norm= 1.0` 이면 합격. norm이 1.0이면 **재정규화 금지**입니다. 다른 값이 나오면 모델이 바뀌었는지 확인하세요. → 시작프롬프트.md §A-2 ①.

> `usage.prompt_tokens`는 **0으로 오보고**됩니다. 토큰 카운트를 신뢰하지 말고 길이 제어는 문자 수 기준을 1차로 쓰세요(시작프롬프트.md §A-2 ②).

> 이 수동 curl 점검은 [02_로컬환경_LMStudio실측.md §5.2](./02_로컬환경_LMStudio실측.md)의 `scripts/env_check.py` 완성 코드로 자동화됩니다. 결과 해석(기대 vs 위험 신호)도 그 문서 §5.3을 참조하세요. 스캐폴딩(§5)에서는 `scripts/env_check.py` 빈 파일만 만들고, 본체는 해당 문서의 코드를 채웁니다.

### 4-3. (선택) 임베딩 모델 교체 — BGE-M3

한국어 검색 품질을 위해 BGE-M3(**1024차원·ctx 8192**, MIT, 한국어 Recall@1 우수) 교체를 권장하나, GGUF 1~2GB **다운로드는 사용자 결정**입니다. (nomic의 한국어 약점 실측치와 prefix 효과 미미 수치는 [02_로컬환경_LMStudio실측.md §4](./02_로컬환경_LMStudio실측.md)에서 다룹니다.)

- **DECISION**(시작프롬프트.md §B-9 #1): **주 사용 언어가 한국어인가** — BGE-M3 교체 권고의 대전제. 한국어 주언어이면 교체 검토 진입.
- **DECISION**(시작프롬프트.md §B-9 #2): BGE-M3 GGUF 다운로드 허용 여부.
- 교체하면 차원이 768 → 1024로 바뀌어 **인덱스 전체 재생성**이 필요하고, BGE-M3는 norm을 보장하지 않을 수 있어 **norm 재측정 후 정규화 분기를 갱신**해야 합니다(§4-2의 스모크 테스트를 BGE-M3 모델명으로 재실행, 기대 `dim= 1024`). → 시작프롬프트.md §B-5. 메타 필드 갱신은 [벡터스토어 스키마](../03_DATABASE/00_벡터스토어_스키마.md) 참조.
- 셋업 단계에서는 **현재 nomic을 그대로 두고** L2에서 정량 비교 후 결정합니다.

### 4-4. reasoning 종료토큰 점검 (BLOCKING — 가장 중요)

세 생성 모델 모두 **`message.content`가 빈 문자열로 오고 실제 답이 `message.reasoning_content`에 담기며**, reasoning이 종료 토큰을 못 만나 무한 사고 → `finish_reason=length`로 잘립니다(한 문장 질문에 **12,972자** 발생). `/no_think`·`enable_thinking=false`·`reasoning_effort=low`·`max_tokens=4000`은 **전부 무효**로 실측됐습니다. → 시작프롬프트.md §B-4·§A-2 ③. (이 대응 방식 자체가 **DECISION** §B-9 #10입니다.)

이 환경 셋업 단계에서 할 일은 **두 가지**입니다.

**(A) 증상 재현 확인** — 생성 호출이 어떻게 깨지는지 직접 봅니다. **(§1 venv 활성 전제. 미활성 시 `python3` 사용.)**

```bash
curl -s http://localhost:1234/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"google/gemma-4-e4b","messages":[{"role":"user","content":"한 문장으로 RAG를 설명해줘."}],"max_tokens":4000}' \
  | python3 -c "import sys,json; c=json.load(sys.stdin)['choices'][0]; m=c['message']; print('finish=',c['finish_reason']); print('content_len=',len(m.get('content') or '')); print('reasoning_len=',len(m.get('reasoning_content') or ''))"
```

**합격 판정(이진, 1차 기준)**: `content_len= 0` **그리고** `reasoning_len > 0` 이면 BLOCKING 이슈가 이 환경에서도 **재현된 것(합격)**입니다.

- `finish= length`는 **부가 신호**일 뿐 합격 필수 조건이 아닙니다. `max_tokens=4000`이 무효라 length 잘림이 자주 재현되지만, **gemma(ctx 65536)는 4000 토큰 내 자연 종료(`finish=stop`)할 수도 있습니다.** 이때도 `content_len=0`이면 **동일 BLOCKING**이므로 재현 실패로 오판하지 마세요.

**(B) LMStudio 프리셋 종료토큰 점검 (근본 대응 절차)** — 코드 폴백(시작프롬프트.md §B-7)보다 근본적인 해결입니다.

| 단계 | 위치 | 점검 항목 |
| --- | --- | --- |
| 1 | LMStudio → **My Models** → 해당 모델 선택 | 모델별 프리셋(설정) 진입 |
| 2 | 모델 설정 → **Prompt / Template** 또는 **Inference** 영역 | reasoning(think) 파서 설정 위치 확인 |
| 3 | reasoning 파서 / **Stop·종료 토큰** | `</think>`, EOS 등 종료 토큰이 올바로 잡히는지 점검·수정 |
| 4 | (A) 재현 테스트 재실행 | **합격 조건: `content_len > 0` AND `finish= stop`** 으로 바뀌면 근본 해결 |
| 5 | 4가 실패하면 | **코드 폴백(§B-7)으로 진행** — 더 이상 프리셋에 매달리지 말 것 |

> **메뉴 위치는 LMStudio 버전마다 다를 수 있어 [확인필요]**입니다. 어느 설정을 어떤 값으로 바꿔야 `finish=stop`이 되는지 구체값은 버전 의존적이라 본 문서가 단정하지 않습니다. 프리셋 점검으로 §4-4(A) 재실행이 `content_len>0 AND finish=stop`이 될 때까지 고쳐지지 않으면 **위 표 5단계대로 코드 폴백(content 빈값→reasoning_content 폴백 + `<think>` 정제 + length 재시도)으로 전환하는 것이 필수**입니다(시작프롬프트.md §B-7). 이 이슈가 해결되기 전에는 "RAG 답변 생성"을 완성으로 보고하지 않습니다.
>
> 운영 단계에서 이 BLOCKING을 어떻게 모니터링·롤백하는지는 [운영 런북](../05_운영_평가/01_운영_런북.md)을 참조하세요.
